### Technical Direction
- **10K objects**: procedural grid of cubes/spheres. Each object has unique material (procedural color, roughness, metallic). Textures are 1x1 RGBA8 (minimal VRAM — focus is on bindless index correctness, not texture resolution).
- **Bindless rendering**: each object's material texture accessed via `BindlessIndex` in push constants or per-instance SSBO. Fragment shader: `Texture2D albedoTex = bindlessTextures[materialData.albedoIndex]`.
- **Memory budget ImGui panel**: per-category bar chart (Geometry, Texture, Staging, etc.), total VRAM used/budget, pressure state indicator (green/yellow/red/black).
- **Residency feedback overlay**: optional debug overlay showing per-object access count as color heat (blue=cold, red=hot). Toggleable via ImGui checkbox.
- **Golden image**: headless render via `OffscreenTarget`, compare with `VisualRegression::CompareGoldenImage`. PSNR > 30 dB.
- **Backend selection**: `--backend vulkan|compat|d3d12|gl|webgpu`. Descriptor buffer path auto-selected on Vulkan Tier1.

## Design Decisions

- **No auto-bindless in CreateTexture**: `ResourceManager::CreateTexture(desc, autoBindless=true)` registers with a null `SamplerHandle{}`, causing Vulkan SEH crash. All bindless registration done manually with a valid sampler via `BindlessTable::RegisterTexture(rhiTex, sampler)`. This is a known API gap — `ResourceManager` needs a sampler parameter or a default sampler for auto-register (Phase 5 cleanup).
- **CPU-side residency simulation**: GPU atomic counters deferred to Phase 6b. `ResidencyFeedback::IncrementCounter()` used for CPU-side simulation.
- **1x1 textures without pixel upload**: GPU texture data upload deferred to Phase 6b streaming. Textures exist as valid bindless entries; GBuffer pass uses material albedo from StandardPBR UBO.
- **100x100 grid = 10K objects**: procedural HSV colors for diversity across full grid.
