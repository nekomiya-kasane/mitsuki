### Technical Direction
- **Counter buffer**: `R32_UINT` storage buffer, `maxResources` entries. Bound as UAV in fragment/compute shaders that access bindless resources. Shader macro: `RESIDENCY_TRACK(resourceIndex)` → `atomicAdd(residencyCounters[resourceIndex], 1)`.
- **Analyze compute shader**: reads counter buffer, produces sorted `{resourceIndex, accessCount}` pairs. Uses parallel prefix sum + compact for non-zero entries, then GPU radix sort by access count (descending). Output to readback buffer.
- **CPU readback**: `Analyze()` dispatches compute, reads back priority list from previous frame (1-frame latency). Returns `PriorityList` = `std::vector<ResidencyPriority>`.
- **Overcommit detection**: if `MemoryBudget::GetPressureState() >= Critical` AND some registered resources have zero access count for N consecutive frames → mark as eviction candidates. `GetEvictionCandidates()` returns resources sorted by staleness.
- **Conditional execution**: if `PressureState == Normal`, skip analysis dispatch entirely (zero GPU cost for unconstrained scenarios).

## Design Decisions

- CPU-side counter tracking in Phase 4 (MockDevice compatible). GPU compute analysis (residency_analyze.slang) is a stub — dispatched in Phase 6b when GPU radix sort is available.
- `IncrementCounter()` added for CPU-side testing — simulates what shaders do via `atomicAdd`.
- Stale frame tracking in `ResetCounters()`: checks each counter before clearing. Zero count increments `staleFrames[i]`, non-zero resets it to 0.
- Conditional execution: `Analyze()` checks `MemoryBudget::GetPressureState()`. Normal = skip entirely. Warning/Critical/OOM = run analysis.
- Priority list sorted by `std::sort` (CPU). Phase 6b replaces with GPU radix sort.
- 11 tests: priority sorting, eviction candidates with stale threshold, conditional skip, max count capping.
