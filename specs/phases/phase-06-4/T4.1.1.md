### Technical Direction
- **Header-only template**: `SlotMap<T>` is header-only for zero-cost generic instantiation. No .cpp file.
- **Dense + sparse design** (Bitsquid/Our Machinery pattern): separate dense data array (for cache-friendly iteration) + sparse index→dense mapping + free-list. O(1) alloc via free-list pop, O(1) free via swap-and-pop + free-list push, O(1) lookup via sparse array.
- **Key layout**: `SlotMapKey` = `{generation:16, index:32}` packed into 48 bits (or 64 bits with padding for alignment). Generation prevents ABA stale access.
- **C++23**: use `std::span` for iteration, `[[nodiscard]]` on all queries, `constexpr` where possible.
- **No VMA/GPU dependency**: pure CPU data structure. Lives in `miki::resource` namespace. Depends only on `<cstdint>`, `<vector>`, `<cassert>`.

## Design Decisions

- Dense+sparse design with swap-and-pop free. `std::vector` for all storage (dense data, dense-to-sparse map, slots, free-list). No custom allocator needed at this scale.
- `SlotMapKey` uses `uint32_t index + uint16_t generation` = 8B with padding. 16-bit generation supports 65535 reuses before wrap — sufficient for resource management (resources are not allocated/freed 65K times per slot in practice).
- Header-only: zero .cpp, zero link-time cost. INTERFACE CMake target.
- `move_constructible` requirement on T: tested with `unique_ptr<int>` (move-only type).
- 15 extra tests beyond spec minimum (spec: 11, actual: 15) — added `FreeInvalidKeyNoOp`, `FreeOutOfRangeIndex`, `SwapAndPopPreservesOthers`, `GenerationWrapsToOne`.
