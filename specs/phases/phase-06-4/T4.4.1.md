### Technical Direction
- **Ring buffer design**: single large `CpuToGpu` buffer (default 64MB). Persistent-mapped via `IDevice::MapBuffer`. Allocations advance a head pointer. Per-frame fence values track which regions are safe to reuse.
- **Per-frame fence tracking**: each frame, `FlushFrame(fenceValue)` records the current head position + fence. `Allocate()` checks if tail region's fence has completed before reuse.
- **Wraparound**: if allocation would cross buffer end, waste remaining bytes (small padding) and wrap to start. If tail hasn't advanced enough → overflow.
- **Overflow fallback**: allocate a temporary one-shot staging buffer. Upload, then destroy after fence. Logged as performance warning.
- **Alignment**: all allocations aligned to `kStagingAlignment` (256 bytes — covers all backend alignment requirements: Vulkan `minStorageBufferOffsetAlignment`, D3D12 constant buffer alignment, etc.).
- **Replaces `StagingUploader`**: Phase 2's `StagingUploader` was one-shot per upload. `StagingRing` amortizes allocation cost across a frame. `StagingUploader` remains for one-time init uploads; `StagingRing` is for per-frame dynamic uploads.

## Design Decisions

- Pimpl: `StagingRing::Impl` holds device ptr, backing buffer, mapped base, head/tail pointers, frame records vector.
- Ring reset: when all frame records are reclaimed AND `head==tail`, reset both to 0 on next `Allocate()`. Prevents phantom wrap calculations from stale tail position.
- `ReclaimCompleted` also resets `tail = head` when frame record list becomes empty (full reclaim = ring empty).
- Overflow returns invalid `StagingAllocation` (nullptr mappedPtr). Caller responsible for fallback. No temporary buffer allocation in this implementation (deferred to ResourceManager layer).
- `BufferUsage::TransferSrc` + `MemoryType::CpuToGpu` for the backing buffer.
