### Technical Direction
- **Auto-register on create**: `CreateTexture(desc, autoBindless=true)` → after `IDevice::CreateTexture` succeeds → `BindlessTable::RegisterTexture(rhiHandle, sampler)` → store `BindlessIndex` in `ResourceEntry`. Same pattern for `CreateBuffer(desc, autoBDA=true)` → `BDAManager::Register`.
- **Auto-unregister on deferred destroy**: in `Tick()`, when a deferred entry reaches `kMaxFramesInFlight` age: (1) `BindlessTable::Remove(entry.bindlessIndex)` if was bindless-registered, (2) `BDAManager::Remove(entry.rhiBufferHandle)` if was BDA-registered, (3) `IDevice::Destroy*(entry.rhiHandle)`, (4) `MemoryBudget::Release(category, size)`.
- **Budget integration point**: `ResourceManager` holds an optional `MemoryBudget*` (set via `SetBudget(MemoryBudget*)`). If set, `Track` on create, `Release` on actual destroy. If null, budget tracking is skipped (for tests without budget).
- **Tick frame index**: `Tick(uint32_t currentFrameIndex)` — compares `currentFrameIndex - entry.frameDestroyed >= kMaxFramesInFlight` to decide when to actually destroy.

## Design Decisions

- Dependency injection via `SetBindlessTable(BindlessTable*)` / `SetBDAManager(BDAManager*)` — nullable pointers. If null, auto-register is silently skipped. This allows tests to run without BindlessTable/BDA.
- `CreateTexture(desc, autoBindless=true)` calls `BindlessTable::RegisterTexture(rhiHandle, SamplerHandle{})` with default sampler. Real sampler binding is the caller’s responsibility (or extended in Phase 6a).
- `CreateBuffer(desc, autoBDA=true)` calls `BDAManager::Register(rhiHandle, desc.size)`.
- Tick auto-unregister: before `DestroyRhiResource`, removes from BindlessTable (if `bindlessIndex.IsValid()`) and BDAManager (if `bdaPointer.IsValid()` and entry is a BufferHandle).
- Budget integration deferred — `SetBudget` not needed yet (MemoryBudget is T4.6.1).
