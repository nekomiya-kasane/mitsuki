### Technical Direction
- **Strategy pattern**: `BindlessTable` internally holds a `std::variant<DescSetStrategy, DescBufferStrategy>` (or type-erased via Pimpl). Feature-detect at Create() determines which strategy.
- **Descriptor buffer**: GPU-mapped `VkBuffer` with `VK_BUFFER_USAGE_RESOURCE_DESCRIPTOR_BUFFER_BIT_EXT`. Each descriptor is a fixed-size byte blob (queried via `vkGetDescriptorSizeEXT`). Register() = memcpy descriptor bytes at `base + index * descriptorSize`.
- **Zero-allocation updates**: no `vkAllocateDescriptorSets` or `vkUpdateDescriptorSets`. Direct memory writes. ~5x faster than descriptor set updates for large table changes.
- **Binding**: instead of `vkCmdBindDescriptorSets`, use `vkCmdBindDescriptorBuffersEXT` + `vkCmdSetDescriptorBufferOffsetsEXT`.
- **Feature detection**: check `VkPhysicalDeviceDescriptorBufferPropertiesEXT` for `descriptorBufferOffsetAlignment`, `maxResourceDescriptorBufferRange`, etc.

## Design Decisions

- **VulkanFeatureMap fix**: `VK_EXT_descriptor_buffer` extension name was missing from the feature mapping table (`NO_EXTS`). Fixed to `EXTS(kDescriptorBufferDevExts)` with implicit dependency on `BufferDeviceAddress` + `DescriptorIndexing`.
- **Conditional pNext chain**: `VkPhysicalDeviceDescriptorBufferFeaturesEXT` only added to pNext chain AFTER `ResolveVulkanFeatures()` confirms the extension is available. Prevents `vkCreateDevice` failure on unsupported hardware.
- **DescBufferStrategy as optional**: `BindlessTable::Impl` holds `std::optional<DescBufferStrategy>` under `#ifdef MIKI_HAS_VULKAN`. Strategy dispatch via `useDescriptorBuffer_` flag. Fallback to descriptor set path is automatic.
- **TryCreate returns NotSupported**: `DescBufferStrategy::TryCreate()` currently returns `NotSupported` because it needs raw VkDevice/VkPhysicalDevice handles from the caller. BindlessTable correctly falls back to descriptor set path. Phase 5 refinement: pass Vulkan handles through BindlessTable::Create() desc or a VulkanDevice accessor.
- **MockDevice fix**: Removed `DeviceFeature::DescriptorBuffer` from MockDevice capabilities — Mock shouldn't claim to support a hardware Vulkan extension.
