### Technical Direction
- **Distinct from RHI Handle**: `ResourceHandle` is application-level (scene objects). `rhi::Handle<Tag>` is GPU-level (Vulkan/D3D12 resources). `ResourceManager` bridges the two.
- **Bit layout**: `[type:4 | generation:12 | index:32 | reserved:16]` = 64 bits. 4-bit type supports up to 16 resource types. 12-bit generation supports 4096 reuses before wrap. 32-bit index supports 4B resources. 16-bit reserved for future (flags, LOD level, etc.).
- **BindlessIndex**: `uint32_t` wrapper. Index into the global descriptor set/buffer. Distinct type to prevent accidental misuse with raw integers.
- **C++23**: `constexpr` construction, `std::bit_cast` for packing, `[[nodiscard]]` on all queries.

## Design Decisions

- Bit layout `[type:4|generation:12|index:32|reserved:16]` MSB-first. Type in highest bits allows fast category filtering via shift+mask.
- `reserved` 16-bit field for future LOD level, dirty flags, or per-resource metadata. Zero cost today.
- 12-bit generation (4095 max) is sufficient — resources are not recycled 4K times per slot in practice. If needed, can be extended by borrowing from reserved.
- `BindlessIndex` as a distinct struct (not `uint32_t` alias) prevents accidental misuse with raw integers — type-safe at zero runtime cost.
- 16 tests (spec: 9) — added `SizeIs8Bytes`, `Equality`, `NullHandleIsZero`, `ZeroIsValid`, `Equality` (BindlessIndex), `SizeIs4Bytes`, `CountFitsIn4Bits`.
