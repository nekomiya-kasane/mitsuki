- **Error model**: `GetGlyphOutlines()` returns `Result<vector<BezierPath>>`. Glyph without outlines (bitmap-only font) → error. `SymbolAtlas::GetSymbol()` returns `GlyphEntry` (always succeeds for known symbols, `.notdef` for unknown).
- **Invariants**: `BezierPath` = closed contour of `PathCommand` segments. Each glyph may have multiple contours (e.g., 'B' has 2: outer + inner). Coordinates in font units (unitsPerEm). Special symbols: pre-defined set of ~110 symbols across 3 categories (GD&T, welding, electrical/surface).

**Key types**:
```cpp
enum class PathCommandType : uint8_t { MoveTo, LineTo, QuadTo, CubicTo, Close };
struct PathCommand {
    PathCommandType type;
    float x1, y1;          // end point (or control point 1 for Quad/Cubic)
    float x2, y2;          // control point 2 (Cubic only)
    float x3, y3;          // end point (Cubic only)
};
struct BezierPath {
    std::vector<PathCommand> commands;
    bool isClosed = true;
};

enum class GdtSymbol : uint8_t {
    Straightness, Flatness, Circularity, Cylindricity,
    ProfileOfLine, ProfileOfSurface, Angularity,
    Perpendicularity, Parallelism, Position,
    Concentricity, Symmetry, CircularRunout,
    TotalRunout, Diameter, // ~30 ISO 1101 symbols
    // ...
};
enum class WeldingSymbol : uint8_t {
    FilletWeld, GrooveV, GrooveU, PlugWeld,
    SpotWeld, SeamWeld, BackWeld, SurfacingWeld,
    // ~25 ISO 2553 symbols
};
enum class SurfaceSymbol : uint8_t {
    RoughnessRa, RoughnessRz, MachiningRequired,
    MachiningProhibited, LayDirectionParallel,
    LayDirectionPerpendicular, LayDirectionCrossed,
    // ~15 ISO 1302 symbols
};
```

### Technical Direction

#### Glyph Outline Extraction

- **FT_Outline → BezierPath**: `FT_Load_Glyph(face, glyphIndex, FT_LOAD_NO_BITMAP)` → access `face->glyph->outline`. Iterate contours via `FT_Outline_Decompose()` with callbacks (`move_to`, `line_to`, `conic_to`, `cubic_to`). TrueType fonts use quadratic Bezier (conic); CFF fonts use cubic. We convert all to cubic for uniform output (elevate quadratic → cubic via standard control point conversion).
- **Coordinate system**: FreeType outline coordinates in font units (26.6 fixed-point). Convert to float. Output in unitsPerEm space. Consumer scales to desired size.
- **Use case**: SVG `<path d="...">` generation. PDF text outline embedding. Phase 15a will consume this directly.

#### Special Symbol Atlas

- **Symbol source**: Engineering symbols not reliably present in standard Unicode fonts. Three approaches (in priority order):
  1. **Dedicated symbol font**: Ship a small OFL-licensed engineering symbol font (or subset of STIX Two Math / Noto Sans Symbols 2). Load via `FontManager`, MSDF-render into atlas.
  2. **SVG-defined symbols**: Define symbols as SVG path data (hardcoded `PathCommand` arrays). Rasterize to MSDF via msdfgen's `Shape` API (no FreeType needed).
  3. **Unicode codepoints**: Some GD&T symbols have Unicode mappings (e.g., ⌀ U+2300, ± U+00B1, ° U+00B0, ⊥ U+22A5, ∥ U+2225). Use these where available in Noto Sans.
- **Phase 2 approach**: Combination. Use Unicode codepoints from Noto Sans where available. For non-Unicode symbols (welding, GD&T frame elements), define as SVG path data and render via msdfgen.
- **Pre-baking**: `SymbolAtlas::Create()` pre-generates all ~110 symbols into resident atlas pages (alongside ASCII/Latin pages in `GlyphAtlas`). Lookup by enum: `GetSymbol(GdtSymbol::Diameter) -> GlyphEntry`.
