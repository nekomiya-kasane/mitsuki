### Technical Direction

- **IMaterial design**: Two layers — (a) **C++ data interface**: `IMaterial.h` defines `MaterialParameterBlock` (CPU struct for UBO upload) and `CreateMaterialUBO()`. `StandardPBR` is a plain C++ struct with parameter values + defaults. No virtual class — material type dispatch is compile-time (shader permutation). (b) **Slang shader interface**: `interface IMaterial { float4 ShadeSurface(SurfaceData surface, LightData light); }` exists only in `.slang` files. `StandardPBR` Slang struct implements this. `SurfaceData`: position, normal, tangent, bitangent, UV. `LightData`: direction, color, intensity. The C++ side and Slang side are linked by shared UBO layout (`static_assert` matching).
- **Anisotropic GGX**: Heitz 2014 paper. NDF: `D_GGX_aniso(NoH, roughnessX, roughnessY, tangent, bitangent)`. Geometry: `G_Smith_aniso(NoV, NoL, roughnessX, roughnessY)`. Fresnel: Schlick. Cost: +2 ALU vs isotropic.
- **Multi-scattering energy compensation (Kulla-Conty 2017)**: Single-scattering GGX loses ~40% energy at roughness=1.0. This is unacceptable for CAD (cast iron, sandblasted surfaces appear too dark). Implement Kulla-Conty multi-scattering compensation: (1) Pre-compute directional albedo LUT `E(cosTheta, roughness)` — 512×512 R16F texture, generated once via compute shader or CPU integration. (2) Pre-compute average albedo `E_avg(roughness)` — 512×1 R16F. (3) In BRDF evaluation: `f_ms = (1 - E(cosTheta_o)) * (1 - E(cosTheta_i)) / (pi * (1 - E_avg))` added as compensation lobe. This matches Dassault Enterprise PBR spec-2025x and Filament's implementation. GPU cost: +1 texture fetch + 3 ALU ops per light.
- **Anisotropic multi-scattering query**: `E_lut` is parameterized by **isotropic roughness**. For anisotropic materials, query with effective roughness `r_eff = sqrt(roughnessX * roughnessY)` (geometric mean). This approximation has <5% error for `anisotropy < 0.8` (validated by Turquin 2019), sufficient for CAD-common brushed metal (`aniso=0.6-0.7`). A precise 4D LUT `(cosTheta, roughnessX, roughnessY, phi)` is impractical. Enterprise PBR spec uses the same geometric-mean approximation.
- **Material UBO**: Per-material uniform buffer with all parameters. Bound at descriptor set 1 (set 0 = per-frame/per-draw constants).
- **Clearcoat top-layer (Phase 2 minimal)**: `StandardPBR` declares `clearcoat` and `clearcoatRoughness` fields. Phase 2 implements the **minimal clearcoat Fresnel top-layer**: `float Fc = F_Schlick(VoH, 0.04) * clearcoat; float attenuate = 1.0 - Fc;` applied to the base BRDF output. Clearcoat specular uses isotropic GGX with `clearcoatRoughness` (no separate GBuffer layer). Cost: ~5 ALU. This ensures the carbon fiber preset (`clearcoat=0.8`) renders correctly — without this, the clearcoat parameter is dead code. Full clearcoat with separate normal map and thickness deferred to Phase 3a.
- **KullaContyLut (naming decision — DECIDED)**: Phase 2 generates **Kulla-Conty energy LUTs** via `KullaContyLut` class (renamed from `BrdfLut` for clarity): `E_lut` (512×512 R16F, cosTheta × roughness → directional albedo) + `E_avg` (512×1 R16F, roughness → average albedo). These are for multi-scattering energy compensation only. Phase 3a IBL will generate a **separate** split-sum GGX BRDF LUT (512×512 RG16F, NdotV × roughness → `(scale, bias)`) via `SplitSumLut` class. The two LUTs serve different purposes and both coexist. File: `src/miki/render/KullaContyLut.h` / `.cpp`.
- **Tangent/bitangent**: `MeshData` needs tangent vectors for anisotropic BRDF. Extend `MeshData` to include optional tangent (`float4`, w=handedness). MikkTSpace for generation from UV. For Phase 2 forward pass, compute tangent from normal in shader (Frisvad frame) if mesh lacks tangent data.
- **No texture maps yet**: Phase 2 uses constant material parameters (no albedo map, no normal map). Texture-mapped materials in Phase 3a+ when render graph provides texture binding infrastructure.

## Design Decisions

- **LUT resolution 64x64** (not 512x512): Sufficient accuracy for Phase 2, small binary (~8KB). Upgradeable in Phase 3a.
- **float16 embedded data**: LUT stored as `uint16_t` arrays in `KullaContyLutData.h`, generated offline by `tools/generate_kulla_conty_lut.py` (1024 hemisphere samples/texel). Avoids runtime computation.
- **Staging buffer pattern**: `CreateKullaContyLut` returns textures + staging buffer. `RecordKullaContyUpload` records `CopyBufferToTexture`. Caller destroys staging after GPU submit. MapBuffer deferred to Phase 3a.
- **Clearcoat integrated into pbr_common.slang**: `ClearcoatLayer()` function returns specular contribution + base attenuation factor. Applied in `standard_pbr.slang`.
- **Simplified evaluate path**: `EvaluateStandardPBR_Simple()` for cases without multi-scattering or tangent input.
