### Technical Direction

- **Descriptor type enum**: `DescriptorType{UniformBuffer, StorageBuffer, SampledTexture, StorageTexture, Sampler, CombinedImageSampler}`. Maps to Vulkan `VkDescriptorType`, D3D12 descriptor range types, GL `glBindBufferRange`/`glBindTextureUnit`, WebGPU `WGPUBindGroupLayoutEntry`.
- **Vulkan path**: `vkCreateDescriptorSetLayout`, `vkCreatePipelineLayout`, `vkAllocateDescriptorSets` from pool, `vkUpdateDescriptorSets`. Descriptor pool managed per-device (auto-grow). `VK_EXT_descriptor_buffer` is **not implemented in Phase 2** — deferred entirely to Phase 4 `BindlessTable`. Phase 2 uses traditional descriptor sets on all backends including Vulkan Tier1.
- **D3D12 path**: Root signature creation from layout. CBV/SRV/UAV descriptor heap. `CopyDescriptors` for updates.
- **GL path**: No descriptor set concept. "Layout" is metadata only. "Set" is a list of bound resources. "Update" records the binding info. On `BindDescriptorSet`, issue `glBindBufferRange`/`glBindTextureUnit` calls.
- **WebGPU path**: `wgpu::Device::CreateBindGroupLayout`, `CreatePipelineLayout`, `CreateBindGroup`. Bind groups are immutable — "update" means recreate. **Immutability mitigation**: maintain a ring of N bind groups per descriptor set (N = `FrameManager::frameCount`, default 2). `UpdateDescriptorSet()` on WebGPU creates a new bind group in the current ring slot instead of mutating. Ring index advances with frame. This avoids per-frame allocation overhead — only the first N frames allocate; subsequent frames reuse ring slots. Bind group pool: pre-allocate 64 bind groups per layout (exponential grow 2× if exceeded). Destruction defers to frame-end fence.
- **MockDevice**: Track handle lifecycle, no real GPU resources.
- **BindDescriptorSet on ICommandBuffer**: New method `BindDescriptorSet(uint32_t set, DescriptorSetHandle, PipelineLayoutHandle)`. Must be added to `ICommandBuffer` and all backend implementations.
- **DestroySampler gap fix**: `IDevice::DestroySampler(SamplerHandle)` is missing from Phase 1a (confirmed: `CreateSampler` exists on all 5 backends + Mock, but no `DestroySampler`). Add as part of this task since Phase 2 needs samplers for E_lut/E_avg textures (T2.4.1) and glyph atlas (T2.7.5). Implement on all 5 backends: Vulkan (`vkDestroySampler`), D3D12 (release sampler descriptor), GL (no-op — GL samplers are implicit), WebGPU (`wgpuSamplerRelease`), Mock (handle tracking).

## Design Decisions

- **Vulkan descriptor pool auto-grow**: `EnsureDescriptorPool()` creates pools with 256 max sets and 256 descriptors per type. New pools created when allocation from existing pools fails. `VK_DESCRIPTOR_POOL_CREATE_FREE_DESCRIPTOR_SET_BIT` enables individual set free.
- **VkImageView gap**: `UpdateDescriptorSet` for `SampledTexture`/`StorageTexture` passes `VK_NULL_HANDLE` for `imageView` because the handle pool stores `VkImage`, not `VkImageView`. Image view management will be added in T2.2.1a (pipeline creation) when real texture sampling is needed.
- **D3D12/GL/WebGPU stubs**: Return `ErrorCode::NotImplemented`. Real implementations will be added when each backend's downstream consumers (T2.2.2, T2.5.1) need descriptor binding.
- **`DestroySampler` gap fix**: Added to all 5 backends + Mock. Vulkan calls `vkDestroySampler`, GL calls `glDeleteSamplers`, WebGPU calls `wgpuSamplerRelease`, D3D12 is no-op (CPU descriptors), Mock tracks handle.
- **Test replaced**: `InvalidBindingFails` from spec replaced with `SimulateErrorOnCreateLayout` — tests error injection on descriptor creation, which is more testable via MockDevice.
