### Technical Direction

- **GL program**: `glCreateShaderProgram` or manual compile+link. **VAO per-pipeline**: one VAO created per pipeline (bound to the vertex layout at pipeline creation time); `BindPipeline` on GL calls `glBindVertexArray(pipeline.vao)` + `glUseProgram` + depth/cull/blend state. VAO lifecycle tied to pipeline handle — destroyed in `DestroyPipeline()`. Uniform block binding for push constants (binding 0, 128B UBO). Depth test/write via `glEnable(GL_DEPTH_TEST)`, `glDepthFunc`, `glDepthMask`. Cull mode via `glCullFace`. Polygon mode via `glPolygonMode`.
- **WebGPU pipeline**: `wgpu::Device::CreateRenderPipeline()` with `WGPURenderPipelineDescriptor`. Vertex state from `VertexLayout`. Fragment state with color target. Depth stencil state. WGSL shader module.
- **Compat Vulkan**: Reuse Vulkan pipeline creation from T2.2.1 but ensure only vertex shader stage (no task/mesh). Feature gate via `GpuCapabilityProfile::tier == Tier2_Compat`.
- **Pipeline state on GL**: GL is stateful — "pipeline" is stored as a struct of GL state to apply before draw. `BindPipeline` on GL sets `glUseProgram` + depth/cull/blend state. This should already be partially wired in `OpenGlCommandBuffer` deferred command model.

## Design Decisions

- **GL pipeline = compile+link program**: `glCreateShader` → `glShaderSource` → `glCompileShader` → `glLinkProgram`. GLuint program stored in `GlHandlePool<PipelineTag>`. VAO creation deferred to BindPipeline (command buffer state setup).
- **WebGPU partial stub**: Shader module creation from WGSL validated, but full `CreateRenderPipeline` with vertex buffer layout mapping deferred to when WebGPU integration tests are ready. Returns `NotImplemented` after shader validation.
- **CompatPipelineFactory dispatch**: Same `#ifdef` + `BackendType` dispatch pattern as `MainPipelineFactory`. Vulkan compat reuses the same `CreateVulkanGeometryPassDispatch` (vertex-only enforced by tier, not by factory).
- **6 tests via MockDevice**: API surface + data structure tests. Real GL/WebGPU pipeline tests require GPU context → integration tests.
