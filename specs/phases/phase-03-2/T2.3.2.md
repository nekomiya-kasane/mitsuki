### Technical Direction

- **DescriptorWriter**: Fluent builder. `DescriptorWriter(IDevice&, DescriptorSetHandle)` → `.WriteBuffer(binding, buffer, offset, size)` → `.WriteTexture(binding, texture, sampler)` → `.Commit()`. Collects `DescriptorWrite` entries, calls `UpdateDescriptorSet` once.
- **Batch write optimization (deferred to Phase 3a)**: Phase 2 `Commit()` directly calls `IDevice::UpdateDescriptorSet()` — no batching infrastructure. Vulkan's `vkUpdateDescriptorSets` natively accepts a span of `VkWriteDescriptorSet`, so a single `Commit()` with N writes is already efficient. Phase 2 material UBOs are constant (no textures), so per-material descriptor sets are written once at init, not per-frame. Batched `FlushAllPending(IDevice&)` deferred to Phase 3a when render graph introduces per-frame transient descriptor set creation at scale.
- **Standard push constant structs**: `ForwardPushConstants{float4x4 model;}` (64B — viewProj is in per-frame CameraUBO, not push constants; see T2.5.1 LightBuffer SSBO design). `TextPushConstants{float4x4 viewProj; float2 screenSize; float2 _pad;}` (80B — padded from 72B to 16-byte alignment for SIMD friendliness). These are convenience structs, not enforced by the API.
- **PipelineLayout push constant range declarations**: ForwardPass pipeline layout declares `{offset=0, size=64, stageFlags=Vertex}`. TextRenderer pipeline layout declares `{offset=0, size=80, stageFlags=Vertex|Fragment}`. Both fit within 128B Vulkan/D3D12/GL/WebGPU limits. Each pipeline's `PipelineLayoutDesc` must explicitly list its push constant range.
- **No descriptor pool management exposure**: Pool is internal to each backend device. Users only see layouts, sets, and writes.

## Design Decisions

- **WriteTexture auto-selects type**: If sampler is valid → `CombinedImageSampler`, otherwise → `SampledTexture`. Avoids caller needing to specify descriptor type.
- **Single Commit()**: No batch/deferred infrastructure in Phase 2. `Commit()` calls `UpdateDescriptorSet()` directly. Phase 3a will add `FlushAllPending()`.
- **TextPushConstants 80B**: Padded from 72B (float4x4 + float2) to 80B with explicit `_pad` for 16-byte alignment. Fits within 128B limit on all backends.
- **ForwardPushConstants 64B**: Only model matrix. viewProj moved to per-frame CameraUBO (descriptor set 0) per T2.5.1 design.
