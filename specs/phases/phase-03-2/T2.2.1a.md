### Technical Direction

- **ShaderHandle storage ‚Äî DECIDED**: `GeometryPassDesc` accepts `ShaderModuleDesc{const void* code, size_t codeSize, ShaderStage stage}` directly (option b). No `ShaderHandle` or `CreateShaderModule` on `IDevice`. Rationale: avoids extra handle type + lifetime management; consistent with Filament (`Program::shader()`) and Wicked Engine (inline bytecode). Backend creates and destroys the shader module internally during `CreateGeometryPass()`. If shader module caching is needed (Phase 3b pipeline cache), it stays internal to the backend.
- **GeometryPassDesc new fields (Phase 2 additions)**: The following fields are added to the existing `GeometryPassDesc` (already defined in `IPipelineFactory.h`): `ShaderModuleDesc vertexShader` (replaces prior `ShaderHandle`), `ShaderModuleDesc fragmentShader`, `CompareOp depthCompareOp` (default `Less`), `BlendState colorBlend` (default opaque, no blend). `CompareOp` enum: `{Never, Less, Equal, LessEqual, Greater, NotEqual, GreaterEqual, Always}`. `BlendState{bool enabled; BlendFactor srcColor; BlendFactor dstColor; BlendOp colorOp; BlendFactor srcAlpha; BlendFactor dstAlpha; BlendOp alphaOp}`. These fields are required by Phase 3a/3b for `CreateShadowPass()` (depth-only, no color blend) and `CreateAAPass()` (post-process blend).
- **Vulkan pipeline**: `VkGraphicsPipelineCreateInfo` with `VK_DYNAMIC_STATE_VIEWPORT`, `VK_DYNAMIC_STATE_SCISSOR`. Dynamic rendering (`VkPipelineRenderingCreateInfo`). Vertex input state from `VertexLayout`. Rasterization state from `CullMode`, `PolygonMode`. Depth/stencil state from `depthTest`, `depthWrite`. Color blend: alpha blend disabled (opaque forward).
- **D3D12 pipeline**: `D3D12_GRAPHICS_PIPELINE_STATE_DESC`. Input layout from `VertexLayout`. Root signature with push constants (root constants at slot 0). Rasterizer state, depth/stencil state. DXIL shader bytecode.
- **Pipeline cache**: Phase 2 does NOT implement persistent pipeline cache (that's Phase 3b). But we do create `VkPipelineCache` in memory for the session.
- **Handle tracking**: Each backend device must track created pipelines for cleanup. Vulkan: `vkDestroyPipeline`. D3D12: `Release()` on PSO.

## Design Decisions

- **ShaderModuleDesc (inline bytecode)**: `GeometryPassDesc` takes `ShaderModuleDesc{code, codeSize, stage}` instead of `ShaderHandle`. Backends create/destroy `VkShaderModule` internally during pipeline creation. No separate handle type or lifetime management needed.
- **CompareOp + BlendState**: Added `CompareOp` (8 values), `BlendFactor` (10 values), `BlendOp` (5 values), `BlendState` struct. Default: opaque (blend disabled), depth compare Less.
- **Vulkan implementation**: Full `VkGraphicsPipelineCreateInfo` with dynamic viewport/scissor, `VkPipelineRenderingCreateInfo` (dynamic rendering, no VkRenderPass), vertex input from `VertexLayout`, rasterizer/depth/blend state mapped via `VulkanUtils.h` converters. Temp `VkPipelineLayout` created inline (empty ‚Äî push constants handled via separate pipeline layout from T2.3.1).
- **D3D12 stub**: Returns `NotImplemented`. Real PSO creation deferred until D3D12 integration tests need it.
- **`#ifdef` dispatch**: `MainPipelineFactory::CreateGeometryPass()` dispatches to `CreateVulkanGeometryPassDispatch` / `CreateD3D12GeometryPassDispatch` behind `#ifdef MIKI_BUILD_VULKAN` / `#ifdef MIKI_BUILD_D3D12` to avoid linker errors when a backend is disabled.
- **Triangle demo (Step 5)**: Deferred to user manual verification ‚Äî requires running with a GPU window.

### Architecture Decision: Method D Resource Wrappers

**Decision**: Adopt Method D (texture-owns-views) for Vulkan resource management.

**Rationale**: View lifetime is always bound to texture lifetime in all miki use cases
(render targets, shadow atlas, GBuffer, cubemap, etc.). Independent ImageView handles
(Method C) would cause API bloat, cross-backend impedance mismatch (D3D12 has no
equivalent), and handle pool pressure. 5/7 major engines (UE5, Filament, Wicked,
bgfx, Metal) use Method D or a variant.

**Implementation (Phase 2 ‚Äî B+ subset of D)**:
- `VulkanTexture{VkImage, VkImageView defaultView, void* allocation}` ‚Äî wrapper struct
- `VulkanBuffer{VkBuffer, void* allocation}` ‚Äî wrapper struct
- `VulkanHandlePool::Entry` reduced to `{NativeType native, uint16_t generation, bool alive}`
- Pool is type-agnostic; resource-specific data lives in the NativeType

**Phase 5+ extension path**: Add `std::unordered_map<ViewDesc, VkImageView>` inside
`VulkanTexture` for 1:N view caching (shadow atlas mip slices, depth-only views, etc.).
No public API changes required.
