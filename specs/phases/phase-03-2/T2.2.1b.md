- **Invariants**: Both copy directions share `BufferTextureCopyInfo{offset, rowPitch, width, height, mipLevel}`. Vulkan requires layout transitions around copies. ### Technical Direction

- **Vulkan**: `vkCmdCopyImageToBuffer` / `vkCmdCopyBufferToImage` with layout transitions (`VK_IMAGE_LAYOUT_TRANSFER_SRC_OPTIMAL` / `VK_IMAGE_LAYOUT_TRANSFER_DST_OPTIMAL`). Fence wait for readback.
- **D3D12**: `CopyTextureRegion` in both directions. Resource state transitions (`D3D12_RESOURCE_STATE_COPY_SOURCE` / `D3D12_RESOURCE_STATE_COPY_DEST`). Fence wait for readback.
- **GL**: Readback via `glGetTextureSubImage` or FBO + `glReadPixels`. Upload via `glTextureSubImage2D`.
- **WebGPU**: `encoder.CopyTextureToBuffer` / `encoder.CopyBufferToTexture`. `mapAsync` for readback.
- **Mock**: No-op. Returns zeros for readback.

## Design Decisions

- **CopyTextureToBuffer added**: The reverse direction (texture→buffer) was missing from Phase 1a. `CopyBufferToTexture` already existed. Both share `BufferTextureCopyInfo`.
- **Vulkan real impl**: `vkCmdCopyImageToBuffer` with matching `VkBufferImageCopy` region struct. Requires texture to be in `VK_IMAGE_LAYOUT_TRANSFER_SRC_OPTIMAL`.
- **D3D12/GL/WebGPU stubs**: Return no-op. Real impl deferred to when each backend’s readback tests need it.
- **ReadPixels stub unchanged**: The stub in `OffscreenTarget.cpp` still returns zeros. Real GPU readback (copy + fence wait + map) requires integration-level testing with a real device, deferred to integration tests.
