- **Error model**: Pure function, no side effects. Returns `CompareResult{bool pass, float rmseNormalized, float ssim, float maxChannelDiff}`. `pass` requires **both** `rmseNormalized < rmseThreshold` **and** `ssim > ssimThreshold`.
- **Invariants**: RMSE normalized to [0,1]. SSIM (Structural Similarity Index) in [-1,1], typically [0.9,1.0] for acceptable matches. Default thresholds: `rmseThreshold=0.05` (5%), `ssimThreshold=0.95`. Input: two RGBA8 pixel spans of identical dimensions. Dimension mismatch → `pass=false, rmse=1.0, ssim=0.0`.

### Technical Direction

- **RMSE formula**: `sqrt(mean(sum((a[i] - b[i])^2 for channels) / (255^2 * 4)))` per pixel, then mean over all pixels. Normalized to [0,1].
- **SSIM formula**: Simplified per-channel SSIM (Wang 2004) with 8x8 block window, then averaged across channels and blocks. `SSIM(x,y) = (2*mu_x*mu_y + C1)(2*sigma_xy + C2) / ((mu_x^2 + mu_y^2 + C1)(sigma_x^2 + sigma_y^2 + C2))` where `C1=(0.01*255)^2, C2=(0.03*255)^2`. SSIM is more sensitive to structural differences (e.g. depth ordering errors, edge shifts) than RMSE which can miss localized artifacts.
- **Header-only**: single `.h` file in test infrastructure, no `.cpp` needed. Inline functions.
- **No external dependencies**: pure arithmetic on `span<uint8_t>`.

## Design Decisions

- **Dual metric (RMSE + SSIM)**: RMSE catches global brightness/color drift; SSIM catches localized structural shifts (depth ordering errors, edge misalignment) that RMSE can miss. Pass requires both.
- **8x8 block SSIM**: Per-channel SSIM computed on non-overlapping 8x8 blocks, averaged. Images smaller than 8x8 fall back to RMSE-only.
- **Header-only**: Single `.h` in `tests/common/`, no `.cpp`. Pure arithmetic on `span<uint8_t>`. Zero external deps.
- **INTERFACE library**: `miki::test_common` provides include path for `GoldenImageCompare.h`.
