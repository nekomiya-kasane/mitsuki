- **Error model**: `GenerateGlyph()` returns `Result<MsdfBitmap>`. Glyph with no outline (whitespace) → returns empty bitmap with valid metrics (not error). Corrupt glyph → error code.
- **Invariants**: `MsdfBitmap` = RGBA8 bitmap (4 channels). Default size: 32x32 px/em (configurable via `MsdfConfig`). `MsdfGenerator` is stateless — each call is independent. Thread-safe for parallel generation.

**Key types**:
```cpp
enum class MsdfPixelFormat : uint8_t { RGBA8, RGBA16F };
struct MsdfConfig {
    uint32_t pxPerEm        = 32;              // MSDF bitmap resolution per em
    double   pxRange        = 4.0;             // distance field range in px (for shader screenPxRange)
    bool     overlapSupport = true;            // handle overlapping contours
    MsdfPixelFormat format  = MsdfPixelFormat::RGBA8;  // Phase 2: RGBA8. Future: RGBA16F for higher precision
};
struct MsdfBitmap {
    std::vector<uint8_t> pixels;  // RGBA8, row-major, top-to-bottom
    uint32_t width;
    uint32_t height;
    double   pxRange;             // carry through for shader
};
```

### Technical Direction

- **msdfgen integration**: MIT-licensed library by Chlumsky. CPU-side generation. We use `generateMTSDF()` (4-channel: R,G,B for multi-channel distance + A for true distance). This provides best quality and enables both `median(r,g,b)` SDF sampling and true-distance outline effects.
- **FreeType outline → msdfgen Shape**: Extract `FT_Outline` from `FT_Load_Glyph` → convert contour points to `msdfgen::Shape` (linear/quadratic/cubic segments). msdfgen's `ext/` module provides `loadGlyph()` which does this directly, but we may prefer our own conversion for control.
- **Edge coloring**: `edgeColoringInkTrap()` (preferred) or `edgeColoringSimple()`. InkTrap handles complex glyphs (CJK with many strokes) better than Simple.
- **Error correction**: Enable `MSDFGeneratorConfig::errorCorrection` with `EDGE_PRIORITY` mode for artifact reduction at corners.
- **Parallelization**: msdfgen supports OpenMP internally. For batch generation (atlas pre-population), we can generate multiple glyphs in parallel. `MsdfGenerator` is stateless and thread-safe.
- **Quality vs SDF**: Single-channel SDF loses sharp corners (e.g., 'W', 'M', CJK strokes). MSDF preserves them via per-channel edge assignment. 4-channel MTSDF adds true distance for outline/shadow effects. Cost: ~0.5ms/glyph at 32x32 on modern CPU.

- [ ] **Step 4.5**: Implement async batch generation (background thread)
      **Files**: `src/miki/text/MsdfGenerator.cpp`, `include/miki/text/MsdfGenerator.h` (internal L)
      `GenerateBatchAsync(span<GlyphRequest> requests, Callback onComplete)` — dispatches `GenerateBatch()` on a **thread pool** (`N = std::thread::hardware_concurrency() / 2`, minimum 2 threads). Results delivered via callback (or `std::future`). Main thread uses placeholder glyph (U+FFFD or empty) for pending glyphs until async generation completes. Double-buffer staging: worker threads write to a staging buffer, main thread copies to atlas page in next `FlushUploads()`. Rationale: CJK first-access page (256 glyphs) at 0.5ms/glyph = 128ms on single thread. With 4 threads: ~32ms = ~2 frames placeholder at 60fps. Single `std::jthread` would take ~8 frames — thread pool significantly improves first-CJK-access experience.
      **Acceptance**: async batch of 256 CJK glyphs completes without blocking main thread; placeholder visible during generation
      `[verify: test]`
