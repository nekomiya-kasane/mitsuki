### Technical Direction

- **Vulkan**: `vkCreateQueryPool(VK_QUERY_TYPE_TIMESTAMP)`, `vkCmdWriteTimestamp2`, `vkGetQueryPoolResults`. `GetTimestampPeriodNs()` from `VkPhysicalDeviceProperties::limits::timestampPeriod`.
- **D3D12**: `ID3D12QueryHeap` with `D3D12_QUERY_TYPE_TIMESTAMP`, `EndQuery`, `ResolveQueryData` to readback buffer, `GetTimestampFrequency` on command queue.
- **GL**: `glQueryCounter(GL_TIMESTAMP)`, `glGetQueryObjectui64v`. `GetTimestampPeriodNs()` = 1.0 (GL timestamps are already in nanoseconds).
- **WebGPU**: `passEncoder.WriteTimestamp` if `timestamp-query` feature available. Feature-detect at device creation. If unavailable, all methods are no-op and `GetTimestampResults` returns zeros.
- **Mock**: No-op. Returns 0 for all timestamp values.
- **QueryPoolHandle**: New handle tag in `RhiTypes.h`. Lifecycle: Create → use in command buffer → GetResults → Destroy.

## Design Decisions

- **Vulkan real impl**: `vkCreateQueryPool(VK_QUERY_TYPE_TIMESTAMP)`, `vkResetQueryPool` on creation, `vkGetQueryPoolResults(VK_QUERY_RESULT_64_BIT)`, `vkCmdWriteTimestamp2(ALL_COMMANDS_BIT)`. `GetTimestampPeriodNs()` reads `VkPhysicalDeviceProperties::limits::timestampPeriod`.
- **D3D12/GL stubs**: Return `NotImplemented`. Real impl deferred to when each backend's ForwardPass (T2.5.1) needs timing.
- **WebGPU graceful degrade**: `CreateTimestampQueryPool` returns `NotSupported`. `GetTimestampResults` fills zeros. No crash path.
- **Mock**: Returns 0 for all timestamps, tracks handle lifecycle. `GetTimestampPeriodNs()` = 1.0.
- **5 tests** (spec asked for 4, added `QueryPoolLifecycle` for handle tracking): CreatePoolValid, MockReturnsZero, PeriodNsNonZero, WriteTimestampRecorded, QueryPoolLifecycle.
