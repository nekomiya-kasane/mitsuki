- **Error model**: `TextRenderer::Create()` returns `Result<TextRenderer>`. `Draw()` is void (batches). `Flush(ICommandBuffer&)` records all batched text. Pipeline creation failure → propagated as error from `Create()`.
- **Cleanup (RAII)**: `TextRenderer` destructor destroys: `PipelineHandle` (text quad pipeline), instance `BufferHandle` (per-frame glyph instance buffer), `DescriptorSetLayout` and `PipelineLayout`, per-page `DescriptorSet`s. All via corresponding `IDevice::Destroy*()` calls. Does **not** own `GlyphAtlas` or `TextShaper` (borrowed references). MockDevice leak detection validates correct cleanup.
- **Invariants**: Each glyph = one instanced quad. Per-instance data: `{float2 posPx, float2 sizePx, float4 atlasUV, uint32_t color_packed, uint16_t pageIndex, uint16_t flags}` = 32 bytes/glyph. Max batch: 65536 glyphs/frame (2MB instance buffer). Screen-space text: pixel-snapped positions (no sub-pixel jitter). World-space text: billboarded or plane-anchored.

### Technical Direction

- **Instanced quad rendering**: No vertex buffer — vertex shader generates 4 corners from `gl_VertexIndex` (0-3 for triangle strip) + instance data. Instance buffer holds per-glyph `{posPx, sizePx, atlasUV, color, pageIndex}`. One draw call per atlas page (sorted by page to minimize texture rebinds).
- **MSDF fragment shader**:
  ```
  float median(float r, float g, float b) {
      return max(min(r,g), min(max(r,g), b));
  }
  float sd = median(msdf.r, msdf.g, msdf.b);
  float screenPxDist = screenPxRange * (sd - 0.5);
  float alpha = clamp(screenPxDist + 0.5, 0.0, 1.0);
  ```
  `screenPxRange` = `pxRange * fontSize / msdfPxPerEm` — scales with rendered size.
- **Screen-space text**: Orthographic projection `[0, screenW] x [0, screenH]`. Pixel-snapped: `floor(pos + 0.5)` to avoid sub-pixel blurring. No depth test, no depth write.
- **World-space text**: Model-view-proj transform. Billboarding: extract camera right/up from view matrix, orient quad to face camera. Depth test enabled, depth write disabled (transparent).
- **Push constants**: `TextPushConstants{float4x4 viewProj; float2 screenSize; float2 _pad;}` — 80 bytes (16-byte aligned, as defined in T2.3.2 `PushConstantLayout.h`). `screenPxRange` is computed per-glyph on CPU and passed via per-instance data (not push constants — it varies per glyph based on fontSize/msdfPxPerEm). Fits in 128B push constant limit.
- **Pipeline state**: Alpha blend (SrcAlpha, OneMinusSrcAlpha). Backface cull off (quads may face either way). MSAA compatible (MSDF naturally anti-aliased).
- **Batching strategy**: `Draw()` accumulates `TextDrawCmd` into internal list. `Flush()`:
  1. Shape each cmd via `TextShaper` → `GlyphRun`
  2. For each glyph, `GlyphAtlas::GetGlyph()` → `GlyphEntry`
  3. Compute instance data (position from GlyphRun offsets + GlyphEntry bearings)
  4. Sort instances by `pageId`
  5. Upload instance buffer via `StagingUploader`
  6. For each page batch: bind page texture descriptor, draw instanced quads

## Steps

- [ ] **Step 1**: Write text vertex and fragment Slang shaders
      **Files**: `shaders/text/text_vert.slang`, `shaders/text/text_frag.slang` (internal L)
      Vertex: expand instanced quad from `gl_VertexIndex`. Fragment: MSDF median + screenPxRange. **Must include screenPxRange clamp + fade-out**: `float spr = max(screenPxRange(), 1.0); float fadeAlpha = saturate(screenPxRange());` — prevents salt-and-pepper noise when world-space text is very far (screenPxRange < 1.0). Also include MSDF/bitmap crossfade band: `float t = smoothstep(8.0, 16.0, effectiveScreenPx); alpha = mix(bitmapAlpha, msdfAlpha, t);` — eliminates hard pop at MSDF↔bitmap threshold. Below 8px → pure bitmap. Above 16px → pure MSDF.
