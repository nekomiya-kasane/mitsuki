- **Error model**: `RichTextLayout::Layout()` returns `Result<LayoutResult>`. Invalid `FontId` in any span → error. Empty spans → empty `LayoutResult` (not error).
- **Invariants**:
  - `RichTextNode` = tree node that is either a **leaf span** (contiguous text with uniform style) or a **composite node** (children arranged by a layout rule). This tree-of-nodes architecture enables nesting: a superscript node contains a child span rendered at 0.7x size + vertical offset, and a `StackNode` (Phase 7b) contains upper/lower children for tolerance stacks like `+0.02/-0.01`.
  - `RichTextSpan` = leaf node convenience alias. A flat `span<RichTextSpan>` input is auto-wrapped into a sequence of leaf `RichTextNode`s, so Phase 2 callers use the same simple flat API.
  - Layout processes nodes left-to-right, recursing into children. Each leaf span shaped independently via `TextShaper`. Style offsets (super/sub, underline) computed from font metrics or explicit `verticalOffset`.
  - Bitmap fallback: fontSize < 8px screen height → use `FT_Render_Glyph(FT_RENDER_MODE_NORMAL)` → 8-bit grayscale bitmap → upload to dedicated atlas bitmap page (R8 format). Crossfade band [8,16]px: both MSDF and bitmap coexist, shader blends. Fragment shader branches on per-instance `flags` bit: MSDF path (4-channel median) vs bitmap path (single-channel alpha).

**Key types**:
```cpp
struct TextStyle {
    bool  bold           = false;
    bool  italic         = false;
    bool  underline      = false;
    bool  strikethrough  = false;
    bool  superscript    = false;   // convenience: sets verticalOffset = +fontSize*0.4, sizeScale = 0.7
    bool  subscript      = false;   // convenience: sets verticalOffset = -fontSize*0.2, sizeScale = 0.7
    float verticalOffset = 0.0f;    // explicit vertical offset in layout units (overrides super/sub if nonzero)
    float sizeScale      = 1.0f;    // explicit size multiplier (overrides super/sub if != 1.0)
};

// Leaf node: contiguous text with uniform style
struct RichTextSpan {
    std::string_view text;
    FontId           fontId;
    float            fontSize  = 16.0f;
    TextStyle        style;
    uint32_t         color     = 0xFFFFFFFF;
};

// Tree node: either a leaf span or a composite with children
enum class RichTextNodeType : uint8_t {
    Span,       // leaf — contains a RichTextSpan
    Group,      // horizontal sequence of children (default)
    Stack,      // Phase 7b: vertical stack (upper/lower tolerance)
};
struct RichTextNode {
    RichTextNodeType                type     = RichTextNodeType::Span;
    RichTextSpan                   span;     // valid when type == Span
    std::vector<RichTextNode>      children; // valid when type != Span
    float                          verticalOffset = 0.0f; // node-level offset (applied to entire subtree)
};

struct LayoutGlyph {
    GlyphInfo  glyph;       // from TextShaper
    GlyphEntry atlasEntry;  // from GlyphAtlas
    float      x, y;        // final position in layout coords
    uint32_t   color;
    bool       isBitmap;    // true if bitmap fallback path
};
struct LayoutResult {
    std::vector<LayoutGlyph> glyphs;
    float totalWidth;
    float totalHeight;
    std::vector<float> lineOffsets;  // y-offset per line (for multi-line)
};
```

**Nesting architecture rationale**: A tree-of-nodes design (vs flat span list) is chosen because:
1. **PMI tolerance stacks** (Phase 7b) require vertically stacked sub-expressions (`+0.02` above `-0.01`) that cannot be expressed as a flat sequence.
2. **GD&T feature control frames** (Phase 7b) contain nested boxes with mixed text and symbols.
3. **Phase 9 `RichTextInput`** needs a document tree for cursor navigation and selection across nested structures.
4. **Backward compatibility**: flat `span<RichTextSpan>` input is auto-wrapped into leaf nodes, so all Phase 2 call sites are unchanged.

### Technical Direction

- **RichText layout pipeline**:
  1. **Recursive layout**: `LayoutNode(node, cursor)` dispatches by `RichTextNodeType`:
     - `Span`: resolve font (bold → bold variant if available, else same font) → shape via `TextShaper` → apply style offsets → atlas lookup → emit `LayoutGlyph`s.
     - `Group`: layout children left-to-right, advancing cursor.x by each child's width.
     - `Stack`: (Phase 7b stub) layout children vertically at the same x, offset by line height.
  2. **Style offset resolution** (per-span):
     - If `verticalOffset != 0` or `sizeScale != 1.0`: use explicit values directly.
     - Else if `superscript`: y += fontSize * 0.4, render at 0.7x fontSize.
     - Else if `subscript`: y -= fontSize * 0.2, render at 0.7x fontSize.
     - **Underline**: thin quad at y = baseline + descender + 1px, thickness = fontSize/14.
     - **Strikethrough**: thin quad at y = baseline + xHeight/2, thickness = fontSize/14.
  3. Atlas lookup per glyph → `GlyphEntry`
  4. Determine MSDF vs bitmap path based on effective screen fontSize
  5. Emit `LayoutGlyph` array
  - **Phase 2 scope**: only `Span` and `Group` node types are implemented. `Stack` returns an error with `ErrorCode::NotImplemented`. Phase 7b activates `Stack` layout.
- **Bitmap fallback implementation**:
  - `BitmapGlyphCache` wraps FreeType rasterizer. Caches rendered bitmaps keyed by `{fontId, glyphIndex, fontSize}`.
  - Bitmap format: 8-bit grayscale (R8). Uploaded to dedicated atlas pages (R8 format, separate from RGBA8 MSDF pages).
  - Fragment shader crossfade (not hard switch): in the 8-16px **crossfade band**, both MSDF and bitmap atlas entries must exist for the glyph. `float t = smoothstep(8.0, 16.0, effectiveScreenPx); alpha = mix(bitmapAlpha, msdfAlpha, t);` Below 8px → pure bitmap. Above 16px → pure MSDF. This eliminates visible pop during continuous zoom. **Rationale (Behdad, State of Text 2024)**: hinted bitmap at <16ppem still outperforms SDF-based rendering; CAD engineering drawings commonly have 8-14px dimension labels where bitmap hinting produces crisper output.
  - Threshold band: configurable, default [8px, 16px]. Below low threshold → bitmap only. Above high threshold → MSDF only. Within band → crossfade.
  - Bitmap cache budget: 2 pages x 512x512 R8 = 0.5MB. LRU eviction same as MSDF pages.
- **Underline/strikethrough rendering**: Emitted as thin instanced quads with a solid white atlas entry (1x1 white texel reserved in resident page). Color from span. Width = span total advance. Height = fontSize/14 (clamped to >= 1px).
- **Thread safety**: Not thread-safe. Main-thread only.
