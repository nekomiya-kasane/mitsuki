- **Error model**: `ShapeText()` returns `Result<GlyphRun>`. Invalid `FontId` → error. Empty string → empty `GlyphRun` (not error). Unsupported codepoint → `.notdef` glyph via fallback chain.
- **Invariants**: `GlyphRun` = contiguous array of `GlyphInfo`. Each `GlyphInfo{glyphIndex, fontId, xAdvance, yAdvance, xOffset, yOffset, cluster}`. `fontId` may differ per glyph (when fallback kicks in). `cluster` maps back to source UTF-8 byte offset for cursor positioning.

**Key types**:
```cpp
struct GlyphInfo {
    uint32_t glyphIndex;  // font-specific glyph ID
    FontId   fontId;      // which font produced this glyph (may differ from request if fallback)
    float    xAdvance;    // horizontal advance in px
    float    yAdvance;    // vertical advance (non-zero for vertical CJK)
    float    xOffset;     // x bearing offset from pen position
    float    yOffset;     // y bearing offset from pen position
    uint32_t cluster;     // source string byte offset
};
struct GlyphRun {
    std::vector<GlyphInfo> glyphs;
    float totalWidth;     // sum of xAdvance
    float totalHeight;    // max ascender - min descender
};
struct ShapingOptions {
    float    fontSize    = 16.0f;
    bool     isRTL       = false;  // override auto-detect
    bool     enableKerning = true;
    bool     enableLigatures = true;
};
```

### Technical Direction

- **HarfBuzz integration**: `hb_ft_font_create(FT_Face)` bridges FreeType → HarfBuzz. One `hb_font_t*` per loaded font (cached in `TextShaper`, keyed by `FontId`). `hb_buffer` is thread-local scratch (reused across calls).
- **Shaping pipeline**:
  1. `hb_buffer_add_utf8(text)`
  2. `hb_buffer_set_direction()` — auto-detected from script, or overridden
  3. `hb_buffer_set_script()` — auto-detected via `hb_unicode_script()`
  4. `hb_buffer_set_language()` — optional, defaults to `hb_language_get_default()`
  5. `hb_shape(font, buffer, features, n_features)` — applies kerning, ligatures, GPOS/GSUB
  6. Extract `hb_glyph_info_t` + `hb_glyph_position_t` → `GlyphInfo` array
- **Script detection**: Per-codepoint `hb_unicode_script()`. For mixed-script strings (e.g., "Hello 世界"), segment by script run, shape each run independently with the correct font, then concatenate.
- **Font fallback chain**: `TextShaper` holds an ordered list of `FontId` per script category. When HarfBuzz produces `.notdef` (glyph index 0) for a codepoint, retry with next font in chain. Default chain: `[user-specified, bundled Noto Sans, system fallback]`.
- **ASCII fast path**: For strings where all codepoints are in U+0020-U+007E and font has 1:1 cmap, skip HarfBuzz entirely. Use pre-computed advance table (cached per font+size). This saves ~0.01ms for HUD text.
- **BiDi**: HarfBuzz handles per-run direction. For full paragraph-level BiDi (UAX #9), we would need FriBidi/ICU — **not in Phase 2 scope**. Phase 2 supports explicit RTL via `ShapingOptions::isRTL` for single-direction runs.
- **Thread safety**: Not thread-safe. Single-owner, main-thread only.

- [ ] **Step 6.5**: Implement ShapingCache (LRU)
      **Files**: `src/miki/text/TextShaper.cpp`, `include/miki/text/TextShaper.h` (internal L)
      LRU cache for shaping results. Key = `hash(string, fontId, fontSize, isRTL)`. Capacity configurable (default 2048 entries — increased from 1024 to accommodate CAD scenes with 500-2000 dimension labels). Cache hit returns pre-computed `GlyphRun` without calling HarfBuzz. Cache invalidated on font change. Rationale: HUD text, fixed PMI labels, and dimension annotations are re-shaped every frame with identical inputs — caching eliminates redundant HarfBuzz calls (~0.05ms/call savings for repeated strings). **Cache statistics**: `GetCacheStats() -> ShapingCacheStats{uint64_t hits, uint64_t misses, uint64_t evictions, size_t currentSize, size_t capacity}`. Exposed for ImGui debug panel in `text_demo` (T2.8.2). Hit rate target: >95% for static CAD scenes after warm-up.
      **Acceptance**: second `ShapeText()` call with identical args returns cached result (verify by call count or timing)
      `[verify: test]`
