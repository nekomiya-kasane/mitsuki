### Technical Direction

- **Shader architecture**: `forward_vert.slang` — transforms vertices by push constant model matrix + per-frame viewProj. Computes normalMatrix in vertex shader: Phase 2 uses `float3x3 normalMatrix = float3x3(model)` (correct for uniform-scale geometry which is all Phase 2 procedural meshes). **Always applies `normalize()` to the output**: `worldNormal = normalize(normalMatrix * inNormal)` — zero visible cost on GPU for already-unit-length vectors, but prevents silent shading corruption if any non-uniform scale is accidentally applied. Phase 3a+ with non-uniform-scale imported meshes switches to CPU-precomputed `inverse(transpose(float3x3(model)))` passed via push constant (48B extra, total 112B ≤ 128B limit). Outputs worldPos, worldNormal. `forward_frag.slang` — imports `standard_pbr.slang`, constructs TBN frame via **Duff-Frisvad-Nayar-Stein-Ling revised ONB** (JCGT 2017, "Building an Orthonormal Basis, Revisited") from worldNormal when mesh lacks tangent attribute (Phase 2 default — tangent data added in Phase 3a+). The revised method fixes the numerical singularity of original Frisvad when normal ≈ (0,0,-1). Evaluates BRDF with lights from `LightBuffer`, outputs color.
- **LightBuffer (UBO, all tiers in Phase 2)**: `LightBuffer` is a **uniform buffer** with `MAX_LIGHTS=4` on **all tiers** in Phase 2 — no tier-specific descriptor layout branching. Phase 2 only uses 1 directional light, so UBO capacity is more than sufficient. This keeps the descriptor set layout identical across all 5 backends, simplifying Phase 2 implementation. **Phase 3a upgrade**: Tier1/2 upgrade to SSBO (`StructuredBuffer<Light>`, capacity N) when render graph provides per-pass resource binding and multiple light types (directional + point + spot) are introduced. Tier3/4 remain UBO with `MAX_LIGHTS=4`. The upgrade only affects descriptor set layout creation for Tier1/2 — `RecordDraws()` internals are unchanged.
- **Descriptor layout (explicit binding table)**:
  - **Set 0** (per-frame): `binding=0: CameraUBO{float4x4 viewProj, float3 cameraPos, float2 nearFar}`, `binding=1: LightBuffer (UBO, MAX_LIGHTS=4, all tiers in Phase 2; Phase 3a upgrades Tier1/2 to SSBO)`, `binding=2: E_lut (SampledTexture, 512×512 R16F)`, `binding=3: E_avg (SampledTexture, 512×1 R16F)`, `binding=4: linearSampler (Sampler)`.
  - **Set 1** (per-material): `binding=0: StandardPBR params UBO`.
  - **Push constants** = per-draw (model matrix only, 64B — normalMatrix computed in vertex shader from model matrix).
  - **Reserved bindings 5-7**: reserved for Phase 3a (environment cubemap, shadow map, irradiance SH buffer). Phase 2 shaders must not use bindings 5-7 in set 0.
  - This layout is forward-compatible with Phase 3a deferred GBuffer fill pass (set 0 grows additional bindings for GBuffer textures).
- **3-step API**: `BeginPass(cmdBuf, renderTarget)` — begins dynamic rendering (Vulkan) / sets render targets (D3D12/GL/WebGPU), binds pipeline, sets viewport/scissor. `RecordDraws(cmdBuf, span<DrawCall>, camera, lights)` — updates per-frame descriptor set, then for each draw: push model matrix → bind material descriptor → draw indexed. `EndPass(cmdBuf)` — ends dynamic rendering. Phase 3a render graph calls only `RecordDraws()`; begin/end managed by graph executor.
- **Draw loop**: Meshes sorted by material to minimize descriptor set switches.
- **DrawCall struct**: `DrawCall{MeshBuffers mesh, MaterialId material, float4x4 model, PipelineHandle pipeline = {}, uint64_t sortKey = 0}`. Defined in `ForwardPass.h`. Used by `RecordDraws(cmdBuf, span<DrawCall>, camera, LightBuffer)`. The `pipeline` field defaults to null handle (= use ForwardPass's own pipeline). Phase 2 ignores this field. **`sortKey`**: 64-bit key encoding `{pipeline:16, material:16, depth:32}` for front-to-back opaque sorting. Phase 2 computes `sortKey` from `{0, materialId, floatBitsToUint(depth)}` — minimizes descriptor set switches (primary) and maximizes early-Z rejection (secondary). Callers may pre-sort or let `RecordDraws()` sort internally. **Phase 3a forward-compat**: when render graph mixes deferred + forward-transparent passes, `pipeline` allows per-draw pipeline override and `sortKey` encodes `{pipeline, material, depth}` for optimal state change minimization.
- **OrbitCamera integration**: Camera from Phase 1a `OrbitCamera`. `Update(deltaTime, input)` → `GetViewMatrix()`, `GetProjectionMatrix()`.
- **Depth buffer**: Format from `CapabilityProfile::preferredDepthFormat()` (D32F on Tier1/2, D24S8 fallback on WebGPU/GL if D32F unavailable). Depth test enabled, depth write enabled.
- **Performance target**: 1000 draw calls (100 tri each) < 2ms GPU time on GTX 1060 equivalent. Measured via timestamp query in Step 4.
- **RenderGraph migration (Phase 3a)**: `ForwardPass` becomes a thin wrapper registered as a render graph node. `RecordDraws()` body is unchanged. `BeginPass/EndPass` are no longer called by user code — graph executor handles pass lifecycle. Migration cost: ~30min (caller change only, zero internal refactor).

## Design Decisions

Standalone structs removed to fix `VUID-VkDeviceCreateInfo-pNext-02830`.
- **hostQueryReset (Vk12 core)**: Enabled in `VkPhysicalDeviceVulkan12Features` for `vkResetQueryPool` used by timestamp queries. 100% driver support on Vulkan 1.2+.
