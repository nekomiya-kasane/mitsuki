- **Invariants**: 1000 procedural cubes in a 10x10x10 grid. Each cube assigned a random material preset from `MaterialRegistry`. Orbit camera (Phase 1a `OrbitCamera`). ImGui overlay: FPS, frame time, backend name, material count. Golden image diff < 5% across backends.

### Technical Direction

- **Scene setup**: 10x10x10 cube grid, spacing 2.5 units. Each cube gets a random material from the registry (cycling through presets). Single directional light (45 degrees down).
- **CLI**: `--backend vulkan|compat|d3d12|gl|webgpu`, `--offscreen`, `--frames N` (default 1 for offscreen). Same pattern as triangle demo.
- **Golden image baseline strategy (two-layer)**:
  1. **Per-backend golden images** (regression detection): each backend has its own baseline PNG committed to the repo. CI compares against same-backend baseline. Detects regressions within a single backend.
  2. **Cross-backend parity test**: any two backend outputs compared pairwise — RMSE < 5%. Detects backend-specific rendering bugs. Vulkan Tier1 is the **reference backend** (first to generate golden images; other backends compared against both their own baseline AND the Vulkan reference).
  Offscreen render → `ReadbackBuffer::ReadPixels()` → `GoldenImageCompare()`. Phase 2 implements real pixel readback (replacing Phase 1a stub).
- **CI jobs**: Add `forward_cubes` offscreen run to each existing CI job (Vulkan, D3D12, UBSAN, Compat, WebGPU).
