- **Error model**: `GetGlyph()` is infallible — cache miss triggers on-demand MSDF generation + insertion. If atlas full + all pages pinned → log error, return `.notdef` placeholder entry. **Color glyph handling (Phase 2)**: if `FT_HAS_COLOR(face)` is true for the requested glyph (emoji, color font), return a pre-cached U+FFFD replacement glyph entry (MSDF-rendered replacement character). Full color emoji support (RGBA bitmap atlas pages) deferred to Phase 7b — see roadmap "Color Emoji & BiDi" component.
- **Cleanup (RAII)**: `GlyphAtlas` destructor destroys all owned `TextureHandle` page textures via `IDevice::DestroyTexture()`. The staging buffer used for uploads (borrowed from `StagingUploader`) is not owned. MockDevice leak detection validates all page textures are properly released.
- **Invariants**:
  - Page size: 1024x1024 RGBA8 (4 MB per page).
  - Page capacity: ~900 glyphs at 32x32 px each (with 2px padding, 30×30 grid).
  - Max pages: **tier-adaptive budget**. Tier1/2 (Vulkan/D3D12/Compat): 8 pages (32 MB). Tier3 (WebGPU): 2 pages (8 MB). Tier4 (GL): 4 pages (16 MB). Configurable via `GlyphAtlasConfig::maxPages`.
  - **Rationale for 1024×1024**: 512×512 pages only fit ~256 glyphs — CJK GB2312 common set (3500+) would require ~14 pages and cause frequent LRU eviction. 1024×1024 fits ~900 glyphs per page; 8 pages = ~7200 glyphs, covering full CJK common set + Latin + symbols with minimal eviction.
  - **Resident pages** (pinned, never evicted): ASCII (U+0020-007E, 95 glyphs), Latin Extended (U+00A0-024F, ~432 glyphs), engineering/GD&T symbols (~80 glyphs). Total: ~607 glyphs = ~1 page (1024×1024 fits ~900).
  - **On-demand pages**: CJK, Hangul, Arabic, Cyrillic, etc. LRU eviction when page budget exceeded.
  - Each page is a GPU `TextureHandle` (RGBA8, 1024x1024). Created via `IDevice::CreateTexture()`.
  - Dirty tracking: per-page dirty flag. Only dirty pages uploaded to GPU per frame.

### Technical Direction

- **Shelf packing**: Classic shelf algorithm. Each page maintains shelves (horizontal rows). New glyph placed on current shelf if fits; else open new shelf. Shelf height = max glyph height in that shelf + 2px padding. Simple, fast, ~85% utilization for uniform-size glyphs (MSDF 32x32).
- **Page lifecycle**:
  1. `GetGlyph(fontId, codepoint)` → cache lookup by `{fontId, codepoint}` key
  2. Cache miss → generate MSDF via `MsdfGenerator`
  3. Find page with space (try resident pages for eligible ranges, else on-demand)
  4. Shelf-pack into page → record `GlyphEntry`
  5. Mark page dirty
  6. `FlushUploads()` called once per frame → upload dirty pages via `StagingUploader`
- **LRU eviction**: Page-level, not glyph-level. Each page has `lastAccessFrame` counter. When allocating new on-demand page and budget exceeded, evict page with smallest `lastAccessFrame`. Resident pages have `lastAccessFrame = UINT64_MAX` (never evicted).
- **GPU texture management**: Each page → one `TextureHandle` via `IDevice::CreateTexture(512, 512, RGBA8)`. Upload via `StagingUploader::Upload()` to texture (requires `CopyBufferToTexture` from T2.2.1). Alternatively, `IDevice::UpdateTexture()` if available.
- **Resident page pre-population (two-stage)**: On `GlyphAtlas::Create()`, use **two-stage initialization** (implemented in Phase 2, not deferred to Phase 7b):
  - **Stage 1 (sync, blocking)**: ASCII 95 glyphs only. Batch-generate via `MsdfGenerator::GenerateBatch()` with thread pool → ~12ms. Upload immediately. HUD text available on first frame.
  - **Stage 2 (async, non-blocking)**: Latin Extended + engineering/GD&T symbols (~512 glyphs). Dispatched via `MsdfGenerator::GenerateBatchAsync()` on thread pool. Main thread renders with ASCII glyphs; remaining resident glyphs appear within 1-2 frames as async batches complete and are uploaded in next `FlushUploads()` call. Placeholder glyph (U+FFFD) used for pending glyphs.
  - **Rationale**: Full sync pre-populate (~75ms with pool) blocks SDK embedding startup unacceptably. Two-stage gives 12ms blocking + 0ms perceived latency for remaining glyphs.
- **Fallback atlas**: If virtual paging not viable (e.g., WebGPU texture limits), fallback to multi-page atlas (4 x 2048x2048 = 16M texels = 64MB RGBA8, ~16K glyph capacity, pre-allocated). Configurable at create time.
- **VRAM budget clarification**: 8 virtual pages x 1024x1024 x 4B (RGBA8) = 32MB total (Tier1/2). Bitmap fallback pages (R8): 2 pages x 1024x1024 x 1B = 2MB. Total atlas VRAM budget: ~34MB default (Tier1/2). Tier3: ~10MB. Tier4: ~18MB.
- **Thread safety**: Not thread-safe. Main-thread only. `MsdfGenerator` calls may be parallelized internally.
