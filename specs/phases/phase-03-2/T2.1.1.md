### Technical Direction

- **Interleaved vertex format**: position (float3) + normal (float3) = 24 bytes per vertex. Single VBO, no separate attribute buffers. This matches the `VertexLayout` already defined in `IPipelineFactory.h`.
- **Procedural mesh helpers**: `MakeCube()`, `MakeSphere(subdivisions)`, `MakePlane()` — return `OwnedMeshData` (owns `std::vector<float3> positions, normals` + `std::vector<uint32_t> indices` internally; exposes `AsMeshData() -> MeshData` returning span views into owned storage). `MeshData` itself remains a borrowed-view struct (`span`-based), while `OwnedMeshData` provides the backing storage. These are in the same translation unit, not a separate file.
- **Index buffer**: always uint32. No uint16 optimization at this stage. `MeshBuffers` can add `IndexType indexType` field later (Phase 6a meshlet needs uint8 local indices).
- **Optional tangent field (forward-compat)**: `MeshData` declares `span<float4> tangents` (w=handedness) as an **optional** field (default empty span). Phase 2 procedural meshes do not populate it — fragment shader uses Frisvad ONB from normal instead. Phase 3a+ imported meshes populate tangents via MikkTSpace. Declaring the field now avoids ABI break when tangent data is added later. `OwnedMeshData` correspondingly has `std::vector<float4> tangents` (empty by default). Interleaved vertex format remains `[pos, normal]` = 24B when tangents are absent; becomes `[pos, normal, tangent]` = 40B when present. `UploadMesh()` checks `tangents.empty()` to select format.
- **Namespace**: `miki::gfx` (not `miki::rhi`) — MeshData is a geometry concept, not hardware abstraction. File path: `include/miki/gfx/MeshData.h`, `src/miki/gfx/MeshData.cpp`.

## Design Decisions

- **Tightly-packed GPU vertex**: `miki::core::float3` is 16B (padded) on CPU. `UploadMesh()` packs position/normal as 3×float (12B each) via `memcpy`, producing 24B/vertex GPU stride. This avoids wasting 8B/vertex of padding in the vertex buffer.
- **Forward-compat tangent field**: `MeshData::tangents` declared but empty for Phase 2 procedural meshes. When non-empty, stride becomes 40B. No ABI break when Phase 3a+ populates tangents.
- **Error cleanup on partial failure**: If index buffer creation fails after vertex buffer was created, `UploadMesh()` destroys the vertex buffer before returning error. Same for staging upload failures.
- **Test replaced**: `UploadMeshNullDeviceFails` from spec replaced with `UploadMeshMismatchedNormalsReturnsError` — tests mismatched `positions.size() != normals.size()` which is a more realistic error path. Null device would require changing the API signature to accept pointers.
