- **Architecture**: Each edge → screen-aligned quad (2 triangles). Mesh shader (T1) or vertex shader (T2/3/4) expands edge to quad with width = lineWeight × DPI scale. Fragment shader computes SDF distance to line center → smooth coverage (antialiased). LinePattern sampled by parametric T along edge. **TAA jitter**: quad vertex positions offset by current TAA jitter for temporal stability.
- **Per arch spec §3 Pass #30**: "HLR Render — Graphics (Task/Mesh or Vert) — Edge buffers, LinePattern SSBO — SDF AA edge color RGBA8 — <1.5ms @5M".
- **IPipelineFactory::CreateHLRPass()**: tier-dispatched — T1 creates mesh shader pipeline, T2/3/4 creates vertex+instanced pipeline. Same EdgeBuffer input.
- **DisplayStyleEdgeConfig**: Parameterizes the SDF shader for all 6 edge-centric DisplayStyles defined in arch spec §5.9.2. Each style configures: edge filter mask (which EdgeTypes to include), line pattern override, color, jitter amplitude, extension length. Styles: Wireframe (all edges, thin), HLR (visible solid + hidden dashed), HLR_VisibleOnly (visible only), Pen (silhouette+boundary+crease, black, jitter), Artistic (silhouette+boundary, soft pencil noise), Sketchy (all edges, jitter+extension). Config stored as push constant or UBO per draw.
- **Reuse**: This SDF render pipeline is shared by Phase 7b iso-parameter lines (#73 contour isolines), Phase 7b sketch renderer (#77), Phase 10 contour plot. The `EdgeRenderer::Render()` API accepts any `EdgeBuffer`-format SSBO + `DisplayStyleEdgeConfig`.
