## Design Decisions

- **Reverse-Z depth convention**: HiZ = 0.0 means far plane (no geometry), HiZ = 1.0 means near plane (opaque). Edge visible when `edgeDepth >= hizDepth - bias`.
- **Adaptive sampling**: `N = clamp(screenLength / 4, 4, 64)` — short edges use fewer samples, long edges up to 64. Mip level selected by `log2(screenLength / N)` for conservative HiZ test.
- **Camera via UBO binding**: Full 4x4 viewProj + resolution passed via CameraUBO (368B) at binding 4, not push constants (which would exceed 128B Vulkan minimum guarantee).
- **Push constants (16B)**: Only `edgeCount`, `hizMipCount`, `depthBias` — minimal per-dispatch data.
- **Whole-edge classification**: Edges classified as fully visible (all samples pass) or fully hidden (all samples fail). Partial visibility writes to visible buffer only. T-interval splitting deferred to T7a1.2.2.
- **Explicit buffer barriers**: Transfer → ComputeShader barriers use `BufferBarrier` with proper `AccessFlags` (not generic memory barriers) to satisfy Vulkan validation.
- **Image layout transitions**: HiZ texture must be transitioned `Undefined → TransferDst → ShaderReadOnly` before compute dispatch.
