## Context Anchor

### This Task's Contract

**Produces**: Per-fragment inside-volume test + stencil capping via `SectionCapPass::ExecuteVolumeClip()`.

- **OBB test**: 6 dot-products per fragment (transform world pos to OBB local space, check all 3 axes within half-extents).
- **Cylinder test**: project world pos onto axis, check distance from axis < radius AND height within bounds.
- **Sphere test**: `length(worldPos - center) < radius`. Trivial cost.
- **Boolean**: AND = both volumes pass; OR = either passes; SUBTRACT = first passes AND second fails.
- **Stencil capping**: Reuses T7a1.5.2 stencil inc/dec logic on volume boundary faces. Cross-hatch fill reuses T7a1.6.1 hatch library.
- **Performance**: 6 dot-products per fragment for OBB — trivial cost per arch spec.

## Implementation Notes

- Volume containment tests use iq-style SDF primitives (branchless `abs()` + comparison for OBB, axial projection for cylinder, distance for sphere).
- Boolean volumes reference children by index in the SSBO array; evaluated as a flat loop with switch on `boolOp`.
- Stencil mark shader now supports dual mode: `clipMode=0` (plane, original) and `clipMode=1` (volume, new). Volume mode discards fragments outside the volume (inverse of plane mode which discards fragments on the negative side).
