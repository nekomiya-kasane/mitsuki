- **Algorithm**: For each surviving edge: `dot(faceNormal_A, viewDir)` and `dot(faceNormal_B, viewDir)` via BDA vertex fetch. Silhouette = sign change. Boundary = single-face. Crease = dihedral > threshold. Wire = explicit. `WaveBallot` subgroup early-out.

- **Output**: `CompactedEdgeBuffer` — tight array of surviving classified edges (prefix sum + scatter). `DispatchIndirect` args for downstream.
- **Performance**: < 1ms @10M edges (pre-cull + classify + compact combined). Workgroup 256.

### Technical Direction

- Per arch spec §3 Pass #28: "HLR Classify — Compute — All — Normals, adjacency SSBO — EdgeBuffer".
- **Pre-cull** (Eurographics 2024 patch-based contour extraction): group edges by meshlet, cull meshlets whose normal cone faces entirely away from view. Reuses meshlet bounding data from Phase 6a.
- Silhouette detection: `dot(N_A, V) * dot(N_B, V) <= 0`. Conservative: also classify near-silhouette edges (dot product < epsilon) for robustness.
- **Subgroup ops**: `WaveBallot` for wavefront-level early-out when no edges in wave are silhouettes.
- Crease threshold: configurable via push constant (default: 30° dihedral angle).
- Center/symmetry edges: detected from TopoGraph cylindrical/symmetric face annotations (Phase 5 TopoGraph carries face type metadata).
- **BDA vertex fetch**: vertex normals read via BDA (buffer device address) from `MeshletVertexData`, consistent with material resolve (arch spec §5.4). No separate normal SSBO binding needed.
- **Stream compaction**: `GpuPrefixSum` (Phase 6a primitive) + scatter. Deterministic edge ordering (prefix sum, not append buffer) per arch spec §5.5 deterministic rendering requirement.

## Design Decisions

1. **GpuTopoEdge 64B**: Stores vertex indices, face indices, face vertex indices (for cross-product normal computation), meshlet ID, edge flags, instance ID. Face normals computed in real-time from vertex positions — avoids storing normals per-face.
2. **3-stage pipeline**: PreCull (meshlet normal-cone) → Classify (per-edge dot-product) → Compact (prefix sum + scatter). Each stage separated by compute barrier.
3. **ByteAddressBuffer vertex fetch**: Uses `ByteAddressBuffer.Load3()` with configurable stride (default 12B = float3 pos). Compatible with existing mesh_geo.slang pattern.
4. **WaveActiveBallot early-out**: If no thread in a wave classifies an edge, skip output writes entirely. Reduces memory bandwidth on smooth meshes.
5. **Conservative silhouette**: Near-silhouette edges (|dot| < 0.001) included to prevent popping artifacts during camera rotation.
