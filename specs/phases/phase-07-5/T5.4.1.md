- **Error model**: `Register()` returns `SystemId`. Circular dependency detected at `Build()` time → returns `ErrorCode::CyclicDependency`.
- **Thread safety**: `Build()` and `Run()` are called from main thread. Systems execute on jthread pool. Component access within systems must be read-only or disjoint-write.
- **Invariants**: systems execute in topological order. Independent systems run in parallel. `stop_token` propagated to all systems for graceful shutdown.

### Technical Direction

- **DAG scheduling via manual declaration**: each `ISystem` declares read/write component sets via `Reads<Ts...>` and `Writes<Ts...>` tag types (manual, not compile-time enforced — consistent with Unity DOTS / Bevy pattern). `SystemScheduler::Build()` constructs dependency DAG from declared access patterns. Topological sort → execution levels. Systems in same level execute in parallel via `std::jthread` pool. Compile-time enforcement rejected: adds complexity disproportionate to benefit for a 30-person team.
- **Thread pool**: fixed-size `std::jthread` pool (`std::thread::hardware_concurrency() - 1` workers). Each system receives `std::stop_token` for cooperative cancellation.
- **Simple initial impl**: Phase 5 systems are few (spatial update, render prep). Full parallel scheduling is infrastructure for Phase 8+ where 10+ systems coexist.

## Design Decisions

- **Pimpl**: `SystemScheduler::Impl` hides DAG data from public header. Move-only.
- **Kahn’s algorithm level-by-level**: produces execution levels where all systems in a level are independent → natural parallel dispatch.
- **Conflict detection**: write→read and write→write conflicts generate DAG edges. Read→read is safe (no edge).
- **jthread per parallel level**: simple but effective. For Phase 5’s 2-5 systems, thread creation overhead (~50µs) is negligible. Phase 8+ with 10+ systems can switch to a persistent thread pool if profiling warrants.
- **ErrorCode::GraphCycleDetected** used instead of plan’s `CyclicDependency` — adapted to existing enum.
- **stop_token from stop_source**: single `stop_source` on Impl, `get_token()` passed to each system. `RequestStop()` triggers cooperative cancellation.
