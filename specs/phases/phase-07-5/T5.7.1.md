- **Error model**: no failure modes — RTE is a coordinate transform, always valid
- **Thread safety**: `RteManager` is single-owner. `GetOrigin()` is read-only — thread-safe for concurrent GPU upload.
- **GPU constraints**: `RteOrigin` is `double3` (24B) for CPU-side storage. Shader uses `float3 cameraRelativePos = worldPos - origin` (cast to float after subtraction).
- **Invariants**: after `SetOrigin(newOrigin)`, all `ToRelative(worldPos)` calls subtract newOrigin. ULP error of float32 result < 0.01mm at 100km from origin.

### Downstream Consumers

  - Phase 10: point cloud positions stored as double3, converted via RTE for rendering

### Technical Direction

- **Progressive re-origin**: when camera moves > threshold (default 10km) from current origin, update origin to camera position. All scene positions re-centered relative to new origin.
- **Double-buffered**: previous frame's origin preserved for motion vector computation (prevWorldPos - prevOrigin). `GetPreviousOrigin()` returns last frame's origin.
- **ULP validation**: `ValidatePrecision(worldPos, origin) -> float` returns max float32 ULP error. Assert ULP < threshold in debug builds.
- **DSFloat**: `struct DSFloat { float hi; float lo; }` — Knuth TwoSum, Dekker TwoProduct. CPU-side utility for validation; GPU-side in Slang shader module.
- **WorldPosition**: `struct WorldPosition { double x, y, z; }` — full-precision CPU position. Converted to `float3` via RTE for GPU.
