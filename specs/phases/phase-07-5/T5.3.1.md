- **Error model**: Query construction is compile-time typed — no runtime errors. `ForEach()` skips entities missing required components.
- **Thread safety**: `ForEach()` is read-only on component data — safe for parallel iteration with read locks. Write operations (Attach/Detach) must not overlap.
- **Invariants**: `Query<All<A,B>>` iterates only entities that have BOTH A and B. `None<C>` excludes entities with C.

### Technical Direction

- **Smallest-pool iteration**: `Query<All<A,B,C>>` iterates the smallest pool among A,B,C, then checks membership in others. O(min(|A|,|B|,|C|)) per query.
- **Compile-time filter**: `All`, `Any`, `None` are tag types; `Query` template specialization generates the correct filter at compile time.
- **ForEach callback**: `Query<All<A,B>>::ForEach([](Entity e, A& a, B& b) { ... })` — references to component data, zero copy.
- **Parallel chunk dispatch**: `ForEachParallel(chunkSize, fn)` splits iteration into chunks dispatched to `std::jthread` pool. Phase 5 uses simple serial `ForEach`; parallel version is infrastructure for T5.4.1 SystemScheduler.

## Design Decisions

- **Header-only**: fully template-based, no `.cpp` needed.
- **Smallest-pool pivot via fold expression**: `SmallestPoolIndex` uses `std::array` + `min_element`; `GetEntitiesFromPool` uses a generic lambda fold to pick the N-th pool's entities span.
- **`AllPoolsExist` guard**: returns early (count=0) if any required pool is unregistered, avoiding null dereference in `HasAll`.
- **Macro comma workaround**: test code uses local variables instead of inline `MakeQuery<All<A,B>>` inside `EXPECT_EQ` to avoid C preprocessor comma-in-template-args issue.
- **Any<> empty = no filter**: `HasAny` with empty parameter pack returns true unconditionally (no restriction).
