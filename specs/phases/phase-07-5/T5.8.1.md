- **Error model**: `IKernel::Tessellate()` returns `Result<TessellationResult>`. `IKernel::Import()` returns `Result<ImportResult>`. Errors propagated via `ErrorCode`.
- **Thread safety**: `IKernel` implementations are NOT thread-safe. `ParallelTessellator` (Phase 7b) handles multi-threaded dispatch by creating per-thread kernel instances.
- **Invariants**: miki never directly calls OCCT. All geometry operations go through `IKernel`. `SimKernel` is always available (no external deps).

  - Phase 8: `Import()` populates CadScene assembly tree

### Technical Direction

- **8 interface groups**: (1) Shape CRUD, (2) Tessellation, (3) Topology Query, (4) Surface/Curve Eval, (5) Boolean, (6) Import/Export, (7) Sketch, (8) Measurement. All pure virtual.
- **TessellationResult**: `{ MeshData mesh, std::vector<uint32_t> triangleToFaceMap, std::vector<EdgePolyline> edgePolylines }` — topology mapping for Phase 7a-2 picking.
- **ImportResult**: `{ std::vector<ShapeId> bodies, AssemblyTree tree, std::vector<PmiEntity> pmi, std::vector<SketchEntity> sketches, std::unordered_map<std::string, std::string> metadata }`.
- **SimKernel**: procedural shapes only (cube, sphere, cylinder, torus). `Tessellate()` returns regular mesh. `Import()` returns single body. No real B-Rep. Always available for tests.
- **KernelFactory**: `Register<T>(name)` + `Create(name) -> unique_ptr<IKernel>`. `SimKernel` auto-registered as "sim".
