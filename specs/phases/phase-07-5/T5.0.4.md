### Technical Direction

- **Free-list design**: maintain `std::vector<uint32_t> freeList_` (LIFO stack). `Allocate()`: pop from freeList_ if non-empty, else bump. `Free()`: push to freeList_.
- **No fragmentation concern**: descriptor heap slots are fixed-size (1 descriptor each), so free-list has zero fragmentation.
- **Capacity**: keep 65536 initial capacity, but now slots are reusable.

## Design Decisions

1. **LIFO free-list for single-slot**: `freeList_` (std::vector<uint32_t>) stores freed single slots. `Allocate(1)` pops from free-list first, then bumps. Multi-slot `Allocate(N>1)` always bumps (free-list stores individual slots only).
2. **Free() pushes individual slots**: `Free(offset, count)` pushes `count` individual slots to the free-list. This allows fine-grained reuse even for ranges originally allocated as contiguous blocks.
3. **Zero fragmentation**: descriptor heap slots are fixed-size, so free-list has zero fragmentation by design.
