### Technical Direction

- **32-bit packed layout**: `[generation:8 | index:24]` = 4 bytes total. Index supports 16M entities (2^24 = 16,777,216). Generation wraps at 255 (8-bit), skipping 0.
- **Free-list recycling**: `EntityManager` maintains `std::vector<uint32_t> freeList_` (LIFO). `Create()` pops from free-list (incrementing generation for that slot) or bumps `nextIndex_`. `Destroy()` pushes index to free-list.
- **Null entity**: `Entity{0}` is null (generation=0, index=0). All valid entities have generation >= 1.
- **Compact representation**: `Entity` is trivially copyable, trivially destructible, suitable for `memcpy` to GPU buffers.
- **Namespace**: `miki::scene`

## Design Decisions

- **`std::vector<bool> alive_`** over bitset: simpler, sufficient for 16M entities (~2MB). Phase 14 can switch to bitset if memory profiling warrants.
- **Generation wraps 255ظْ1 (skip 0)**: ensures null sentinel (gen=0) is never reused. 255 reuse cycles per slot before any possibility of ABA ظ¤ far exceeds CAD editing patterns.
- **ForEachAlive iterates all slots**: O(capacity), not O(alive). Acceptable because capacity grows monotonically and typical CAD scenes have <100K entities. Phase 14 can add a dense alive-list if iteration becomes bottleneck.
