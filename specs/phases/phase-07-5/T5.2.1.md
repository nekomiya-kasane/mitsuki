- **Error model**: `Get(entity)` returns `T*` (nullptr if not attached). `Attach()` overwrites if already attached.
- **Thread safety**: NOT thread-safe. Single update thread. Read-only parallel iteration via Query (T5.3.1).
- **GPU constraints**: Dense array layout ensures contiguous component data — Phase 6a `SceneBuffer` can bulk-iterate via `ForEachWithEntity()` for GPU upload.
- **Invariants**: `Attach(e, comp)` → `Get(e) != nullptr`. `Detach(e)` → `Get(e) == nullptr`. Destroy entity → all components auto-detached.
- **Design note**: This is a **Sparse Set** design (each type gets independent dense array + sparse index), NOT true Archetype chunking (flecs/Unity DOTS). Sparse Set is simpler and sufficient for CAD scenes where most entities share the same component set. True Archetype chunking is a Phase 14 scale optimization if profiling shows benefit.

### Technical Direction

- **Sparse Set design**: each `ComponentPool<T>` stores a dense `std::vector<T>` + `std::vector<Entity>` (parallel entity array) + sparse `std::vector<uint32_t>` (entity index → dense index mapping). Swap-and-pop on remove for O(1) and contiguous memory.
- **Type erasure**: `ComponentRegistry` maps `std::type_index` → `IComponentPool*` (type-erased base). `GetPool<T>()` does `static_cast`.
- **Auto-detach on entity destroy**: `EntityManager` notifies registry → each pool checks and detaches.
- **ForEachWithEntity**: `ComponentPool<T>::ForEachWithEntity(Fn(Entity, T&))` — iterates dense array with parallel entity array. Phase 6a SceneBuffer sync uses this to build `GpuInstance[]` from Transform+Mesh components.
- **float3_packed evaluation**: assess whether `struct float3_packed { float x,y,z; }` (12B, no padding) improves memory density for position/normal component arrays. Decision during implementation.

## Design Decisions

- **Fully header-only**: `ComponentPool<T>` is a class template — no `.cpp` needed. `ComponentRegistry` uses `std::type_index` + `unique_ptr<IComponentPool>` for type erasure.
- **Sparse array grows lazily**: `EnsureSparse(idx)` resizes only when a new entity index exceeds current size. No upfront allocation.
- **Attach overwrites**: re-attaching same entity replaces component value in-place (no detach+reattach cycle). Keeps dense index stable.
- **float3_packed evaluation**: deferred — current `Position{float x,y,z}` is 12B naturally (no padding needed for SOA since each field accessed independently). GPU struct padding is a separate concern (GpuInstance uses alignas(16)).
