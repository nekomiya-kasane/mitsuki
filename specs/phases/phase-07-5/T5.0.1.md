## Design Decisions

1. **Default optional Tier1 features in CreateOwned()**: `VulkanDevice::CreateOwned()` now merges Tier1 features (MeshShader, RayTracing, DescriptorBuffer, etc.) into optional features before resolving. This ensures extensions are enabled at `vkCreateDevice` when hardware supports them, making `vkGetDeviceProcAddr` resolve extension function pointers correctly.
2. **BindlessTable descriptor buffer routing**: `RegisterTexture/Buffer/StorageTexture` and `Remove` now branch on `useDescriptorBuffer_` to route through `DescBufferStrategy::WriteTexture/WriteBuffer/WriteStorageTexture/ClearSlot` instead of `UpdateDescriptorSet`. `AllocateSlot` grow path uses `descBuffer_->Grow()` when on descriptor buffer path.
3. **PopulateCapabilities hardware query kept as-is**: The hardware feature detection in `PopulateCapabilities()` adds features to `enabledFeatures` based on VkPhysicalDevice queries. With the `mergedOptional` fix, `resolvedFeatures_` now correctly includes `DescriptorBuffer` when hardware supports it, so the hardware query and resolved features are consistent.
