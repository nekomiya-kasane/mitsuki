### Technical Direction

- **SAH (Surface Area Heuristic) build**: top-down recursive split. Cost function: `C_split = C_trav + p_L * n_L * C_isect + p_R * n_R * C_isect` where p = surface area ratio. Binned SAH (8-16 bins) for O(N log N) build.
- **Flat array storage**: `std::vector<BvhNode>` with implicit indexing. Left child = `2*i+1`, right = `2*i+2`, or explicit child indices for better cache. Leaf nodes store AABB + entity index range.
- **Iterative traversal**: stack-based traversal (no recursion). Stack depth <= 64 (tree depth). Both frustum and ray traversal are iterative.
- **Refit API**: `Refit(span<uint32_t> dirtyLeafIndices)` — bottom-up AABB expansion for moved entities. O(depth * |dirty|). Full rebuild only when refit quality degrades.
- **BvhNode layout**: `{ AABB bounds; uint32_t leftChild; uint32_t rightChild; uint32_t firstPrimitive; uint32_t primitiveCount; }` — 32B + 16B = 48B per node, pad to 64B for GPU alignment.

## Design Decisions

- **Per-primitive AABB storage**: original AABBs stored in `primitiveAABBs_` vector to enable precise per-primitive testing in leaf nodes (multi-prim leaves). Without this, leaf bounds union causes false positives in frustum/ray queries.
- **Binned SAH (16 bins)**: O(N log N) build with 16 SAH evaluation bins per axis. `kMaxLeafPrims = 4` balances tree depth vs leaf iteration cost.
- **Iterative traversal**: stack-based (vector reserve 64), no recursion. Both frustum and ray queries use the same pattern.
- **Refit strategy**: update stored primitive AABBs → recompute leaf bounds → bottom-up sweep all internal nodes. O(N) worst case, but simple and correct.
