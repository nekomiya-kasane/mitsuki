# T7a2.12.1 — Texture Projection Modes (Triplanar/Box/Sphere/Cylinder/Decal)

**Phase**: 12-7a2
**Component**: Texture Projection Modes
**Status**: Not Started
**Effort**: M (1-2h)

## Dependencies

None (L0 task).

## Scope

Per arch spec and roadmap: `MaterialParameterBlock.projectionMode` field (already defined in Phase 2) selects UV computation method. Material resolve compute (T1) and forward fragment (T2/3/4) calculate UV from world position + projection mode.

- **UV (default)**: Standard mesh UV coordinates. No extra cost.
- **Triplanar**: 3 axis-aligned texture samples blended by surface normal weights. `weight = abs(normal)^blendSharpness`. Cost: 3× texture reads + blend = ~6 ALU + 2 extra tex.
- **Box**: 6-face projection, select face by dominant normal axis. 1 tex read per pixel.
- **Sphere**: Spherical UV from `atan2(pos.z, pos.x)` and `acos(pos.y / length(pos))`. ~4 ALU.
- **Cylinder**: Cylindrical UV from `atan2(pos.z, pos.x)` + `pos.y / height`. ~3 ALU.
- **Decal**: OBB projection matrix (Phase 7a-2 decal projector, separate task in future).

### Technical Direction

- Branch on `projectionMode` in material resolve shader. `if (projectionMode == 0) return meshUV;` — zero cost for default UV path.
- World position available from VisBuffer BDA vertex fetch (T1) or from vertex shader interpolation (T2/3/4).
- Per arch spec: "+2 ALU per non-UV pixel (branch on projectionMode, skip if UV)".
- Specialization constant optimization: if scene has no non-UV materials, the branch is compiled out entirely.
