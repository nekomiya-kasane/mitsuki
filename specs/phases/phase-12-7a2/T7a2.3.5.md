## Scope

Per arch spec §(2) Edge/Vertex Proximity Tolerance:

VisBuffer provides pixel-exact selection (zero tolerance). CAD users need 3-10 pixel tolerance for edge/vertex proximity, with priority: Vertex > Edge > Face.

**Primary approach — On-demand neighborhood sampling** (pick-time only, zero per-frame cost):
1. Read VisBuffer in N×N neighborhood (default N=21, 10px radius, 441 pixels).
2. Extract unique `{instanceId, primitiveId}` pairs.
3. For each primitiveId → `TopoGraph::PrimitiveToFace()` → `FaceToEdges()`.
4. Compute screen-space distance from click to each edge's projected polyline.
5. Return closest within tolerance: Vertex (<5px) > Edge (<10px) > Face (fallback).

Cost: ~0.02ms (441 VisBuffer reads + CPU distance computation). Zero per-frame GPU overhead.

**Optional JFA enhancement**: For real-time hover-highlight of nearest edge, per-frame edge SDF via Jump Flooding Algorithm. 12 passes at 4K, ~1.2ms/frame. Only activated when PickLevel::Edge/Vertex mode is active.
