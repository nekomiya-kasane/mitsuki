## Scope

User draws arbitrary screen-space polygon (freehand lasso or click-to-place vertices). GPU generates per-pixel mask via winding number compute.

Per arch spec "(3) Lasso Mask Performance":
- **Douglas-Peucker simplification**: Reduce raw polyline (500+ mouse samples) to 20-50 vertices on CPU. Tolerance: 2 screen pixels.
- **Bounding-box dispatch clip**: Compute lasso AABB on CPU, dispatch only over AABB region (10-30% of screen).
- **Winding number compute**: Per-pixel test against polygon SSBO. O(pixels × vertices). 4K × 30 verts × 20% coverage ≈ 480M iterations, <0.5ms @20 TFLOPS.
- **Stencil fallback (T1/T2)**: For >100 vertex polygons after simplification, use GPU rasterization: triangle fan + stencil odd/even = inside. Faster than compute for high vertex counts.
- **Handles concave polygons**: Winding number test is robust for any simple polygon (non-self-intersecting).

Both no-drill and drill paths share the same mask generation.
