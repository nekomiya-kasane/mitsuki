## Scope

Per arch spec §7.2 and §3 Pass #3:

**3D froxel grid**: `ceil(w/64) × ceil(h/64) × 32` depth slices. Depth distribution: logarithmic (Reverse-Z). Each froxel = AABB in view-space.

**Algorithm** (light-centric assignment):
1. Per-light compute: project light's influence AABB to screen-space → determine which froxels overlap.
2. For each overlapping froxel: `atomicAdd` light index to froxel's light list.
3. Output: `ClusterGrid` SSBO — per-froxel `{uint lightIndices[MAX_PER_CLUSTER], uint count}`.

**Dispatch**: 1 workgroup per light. Each light tests overlap against all froxels in its screen-space footprint. For point/spot lights, the footprint is bounded by the projected AABB of the light's range sphere. For directional lights, all froxels are lit.

**Performance**: <0.1ms @1K lights, <0.3ms @4K lights.

**Tier fallback**: T2/3/4 use CPU-sorted UBO array (no GPU pass). `IPipelineFactory` dispatches.

### Technical Direction

- Per Granite blog (Maister 2020): light-centric assignment via compute is more efficient than froxel-centric for >256 lights. For <256 lights, either approach works.
- MAX_PER_CLUSTER = 256 (Tier1). Overflow: excess lights silently dropped per froxel (sorted by distance, farthest dropped first).
- Froxel AABB computation: `zNear * pow(zFar/zNear, slice/numSlices)` for logarithmic depth.
