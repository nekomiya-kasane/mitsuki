## Scope

Drill box/lasso selection finds ALL instances including fully occluded ones (VisBuffer only has front layer). Per arch spec Pick Architecture "(1) Drill Box/Lasso — Three-Stage GPU Volume Culling":

- **Stage 1 — Instance cull (compute)**: Project each `GpuInstance.boundingSphere` AABB to screen-space. Test: 2D overlap with selection bbox AND depth range intersects `[near, far]`. Output: `CandidateInstanceList` (atomic append). ~0.1ms @100K instances.
- **Stage 2 — Meshlet cull (compute)**: For surviving instances, project meshlet bounding spheres. Test: circle overlaps selection mask. Output: `CandidateMeshletList`. ~0.3ms @1M meshlets.
- **Stage 3 — Triangle collect (compute)**: For surviving meshlets, project 3 vertices per triangle. Test: any vertex inside mask OR triangle bbox overlaps mask. Atomic append `{instanceId, primitiveId}` to HitBuffer. ~0.5ms @100K surviving triangles.

**Why not RT**: RT ray query is optimal for single-point drill (1 ray), but area selection would need millions of rays. 3-stage culling is O(instances + meshlets + triangles) regardless of selection area size.

**Tier compatibility**: All 3 stages are pure compute — works on all tiers including WebGPU/OpenGL.
