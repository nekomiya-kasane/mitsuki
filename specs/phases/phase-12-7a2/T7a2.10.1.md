# T7a2.10.1 — Motion Blur Compute (McGuire 2012 Tile-Based Directional Blur)

**Phase**: 12-7a2
**Component**: Motion Blur
**Status**: Not Started
**Effort**: M (1-2h)

## Dependencies

None (L0 task).

## Scope

Per arch spec §3 Pass #51:

- **Algorithm**: McGuire & Mara 2012 "A Reconstruction Filter for Plausible Motion Blur". Tile-based: (1) compute per-tile max velocity from GBuffer motion vectors (RG16F), (2) per-pixel gather along velocity direction, weighted by depth similarity.
- **Tile size**: 16×16 pixels. Max velocity tile stored as RG16F half-res image.
- **Gather**: 9-15 samples along velocity direction per pixel. Weight = depth similarity × distance-from-center falloff.
- **Tier dispatch**: T1/T2 compute only. T3/T4: disabled (no motion vectors available without TAA).
- **Activation**: Only during turntable/flythrough/kinematic animation. Disabled during interactive editing (mouse drag). Conditional RG node.
- **Budget**: <1.0ms @4K.

### Technical Direction

- Motion vectors from GBuffer RT2 (RG16F, Phase 3a). Already populated by material resolve and GBuffer forward pass.
- Tile max velocity: compute pass reduces 16×16 block → single max velocity. This determines per-tile blur kernel size.
- Skip optimization: tiles with max velocity < 0.5px → no blur (pass-through).
- Temporal stability: jitter velocity by blue noise to break banding.
