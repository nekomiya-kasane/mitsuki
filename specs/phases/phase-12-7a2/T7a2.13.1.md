**Component**: Material Graph

## Dependencies

None (L0 task).

## Scope

Per roadmap: `MaterialGraphCompiler` — node-based material authoring compiled to Slang shader permutations at edit time (not runtime).

- **Node types**: noise (Perlin/Simplex/Worley), checker, gradient, wood grain, carbon fiber, marble, mix, multiply, remap, constant color/float, texture sample.
- **Output**: `MaterialParameterBlock` fields (albedo, roughness, metallic, normal, emission). Each output port maps to a specific MPB field.
- **Compilation**: Graph → DAG → topological sort → emit Slang code per node → compile to `ShaderPermutationKey` (64-bit bitfield from Phase 1a `PermutationCache`). Runtime cost: zero extra vs hand-authored shader.
- **Serialization**: Graph stored as JSON. Round-trip: load → edit → save → recompile.
- **Editor**: ImGui node graph UI deferred to Phase 9. Phase 7a-2 delivers the compiler + programmatic API only.

### Technical Direction

- Per arch spec §6.2: "VGPR control via Vulkan Specialization Constants" — material graph compiled permutations use `[[vk::constant_id(N)]]` booleans to prune dead code.
- Node evaluation order: topological sort of DAG. Each node emits a Slang function call. Inputs are wired as function parameters.
- Graph validation: detect cycles (error), unconnected outputs (warning), type mismatches (error).
- Per arch spec: "Compiled material → ShaderPermutationKey. Runtime cost: zero extra vs hand-authored."
