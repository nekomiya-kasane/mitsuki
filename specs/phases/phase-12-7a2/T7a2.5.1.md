## Scope

Activate the RTAO stub from Phase 3b. Reuse BLAS/TLAS built for picking. Per arch spec §3 Pass #17:

- **Algorithm**: `VK_KHR_ray_query` in compute shader. Per-pixel: reconstruct world position from GBuffer depth + normals. Trace 1 short-range ray (random hemisphere direction, max distance ~1m). Hit = occluded. Output: R8_UNORM AO texture.
- **Temporal accumulation**: 8-frame exponential moving average (EMA) for noise reduction at 1spp. Requires motion vector rejection (discard history when pixel moves significantly).
- **Fallback**: When RT hardware unavailable (`!HasRayTracing()`), silently use GTAO (Phase 3b) instead. Zero code change needed at call site — `IPipelineFactory::CreateAOPass()` dispatches to RTAO or GTAO based on tier.
- **Budget**: <2ms @4K (full-res, 1spp + temporal).
- **Integration**: Deferred resolve (#18) reads AO texture at binding slot — same slot for RTAO and GTAO outputs.

### Technical Direction

- Per arch spec §3: "RTAO — Compute (ray query) — T1 — BLAS/TLAS, depth — R8 AO (1spp+temporal) — <2ms".
- Ray length: short-range (<1m) for ambient occlusion, not global illumination.
- Hemisphere sampling: cosine-weighted, blue noise offset per pixel per frame for temporal stability.
- BLAS/TLAS from pick system (T7a2.2.1-2.2) — zero additional acceleration structure cost.
